# 10. 이주(Migration) 매니페스트 — 원본 → 새 폴더

> **목적**: AI가 원본 폴더를 참조하면서 새 폴더로 **파일을 옮겨 담는** 작업을 헷갈림 없이 자동 수행하게 한다.
> 무에서 생성이 아니라 **이주(migration)**: 원본을 보면서 ① 그대로 복사, ② HTML5로 변환, ③ 제외 — 세 가지로 처리한다.
> 세부 구현은 원본 파일이 진실 공급원이고, 본 매니페스트는 "무엇을 어디로, 어떻게" 분류한 지도다.

## 범례 (Legend)

| 표기 | 의미 | 처리 방법 |
|:----:|------|-----------|
| 📋 **COPY** | UI5 무관. **원본 파일 그대로 복사** | 바이트 단위 복사. 수정 없음 |
| 🔧 **TRANSFORM** | SAP UI5로 화면을 그리는 부분. **원본을 참조해 HTML5+CSS+바닐라 JS로 재작성** | 로직 흐름은 원본 유지, UI 생성 코드만 교체 |
| 🧩 **SPLIT** | 한 폴더 안에 COPY 대상과 TRANSFORM 대상이 섞임 | 하위를 나눠 각각 처리 |
| 🚫 **EXCLUDE** | 개발용 문서/백업/미사용. **새 폴더에 가져오지 않음** | 복사 안 함 |
| 🟡 **KEEP-UI5** | 예외적으로 UI5를 유지(변환하지 않음) | 원본 그대로 복사하되 UI5 SDK 로딩 필요 |
| 🆕 **NEW** | 원본에 없는 **신규 산출물**. 변환으로 새로 생성 | 시안·사양 문서를 보고 새로 작성 |

> 🟡 KEEP-UI5 는 **디자인 미리보기 iframe** 한 곳뿐이다(사용자가 만드는 "대상 앱"을 UI5로 렌더링하므로). → [05_디자인영역.md](05_디자인영역.md)

## 7. ⚠️ 실전 추가 주의 (변환 중 확인된 것 — [11 플레이북](11_컨버전_플레이북_처음시작.md) 참조)

1. **대형 design 파일은 부팅 preload에 넣지 말 것.** `uiDesignArea.js`(6.8천줄) 등은 원래 WS20 진입 시 `getScript`로 **지연 로드**되는 파일. 부팅 스크립트 목록에 추가하면 **앱 전체가 깨진다**(모델 미생성 → `fnWsStart` 중단). → 지연 로드 또는 필요한 함수만 1:1 이식.
2. **부팅 의존성 폐쇄(closure)를 먼저 복사.** 메인프레임 공통(`resources/index.js`)이 `ws_util.js`/`usp/ws_usp_util.js`/`ws_log.js`/`ws_trycatch.js`를 require하고, intro.js가 `u4a-intro`/패턴(`usp/pattern`)을 require한다 → 부팅 셸 단계에서 이들 **기반 파일을 함께 복사**하지 않으면 부팅이 멈춘다.
3. **WS 스크립트는 로드 시점에 UI5를 즉시 실행하지 않는다(함수 정의만).** 따라서 미변환 WS20/WS30/design 파일도 **로드만 되면 안전**하고, 해당 UI5 함수는 호출되기 전까진 문제없다. (호출 지점만 가드/변환하면 됨)
4. **`ws_settings.json`의 UI5 섹션은 메인프레임용.** 미리보기 iframe의 UI5는 별도로 서버에서 받으므로(KEEP-UI5), 메인프레임 UI5 제거 후에도 미리보기는 동작한다.

## 8. 실행 사양 ↔ 매니페스트 연결 (메타)

> 본 매니페스트는 "무엇을 어디로, 어떻게 옮기는가"의 **분류 지도**이고, 실제 변환 절차·수용 기준은 각 기능 문서의 **"AI 컨버전 실행 사양"** 절에 있다. 이 둘은 짝이다: 매니페스트가 산출물을 **추적**하고, 실행 사양이 그 산출물을 **생성**한다.

### 8.1 각 기능 문서의 실행 사양 절

### 8.2 추적 규칙

- 각 실행 사양의 "산출물 파일 트리"가 만들어내는 파일은 본 매니페스트의 해당 영역 표 항목과 **1:1로 대응**해야 한다. 실행 사양이 새 파일을 추가하면 매니페스트에도 항목을 등록한다(드리프트 금지).
- **테마 산출물(`theme/tokens.css`·`shell.css`·`theme-api.js`·`themes/*.css`)은 횡단 산출물**이다. 원본에 없으므로 🆕 NEW 로 표기하며, 모든 화면 CSS 가 이 토큰을 소비한다. 단일 출처는 [12](12_테마_컨버전_전략.md) 이고, 매니페스트는 그 산출물의 **존재만 추적**한다(사양 본문은 12번에 둔다).
- 5절 단계별 진행은 변환 **순서**, 각 기능 문서의 실행 사양은 변환 **방법**이다. 충돌 시 방법은 기능 문서를, 무엇을 옮길지는 본 매니페스트를 따른다.
