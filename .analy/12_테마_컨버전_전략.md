## 1. 문제 정의

UI5 에는 **테마**(`sap_horizon` 등)라는 컨셉이 내장되어 있다. 순수 HTML5 로 컨버전하면 이 컨셉이 사라지는 것처럼 보이지만, 실제로는 **테마 개념이 없어지는 게 아니라 구현 주체가 바뀌는 것**이다.

| UI5 테마 구성요소 | HTML5 등가물 |
|---|---|
| 테마 파라미터 (`@sapBrandColor` …) | **CSS Custom Properties** (`--accent: …`) |
| `applyTheme(name)` | `document.documentElement.dataset.theme = name` |
| `Parameters.get("...")` | `getComputedStyle(el).getPropertyValue("--...")` |
| `attachThemeChanged(fn)` | `CustomEvent` 구독 |
| 테마별 컴파일된 CSS | (컴포넌트 CSS 1벌) + (테마별 **색** 토큰 세트) |

즉 HTML5 컨버전에서 테마 = **CSS 변수 세트 + 그것을 소비하는 컴포넌트 CSS + 얇은 전환/조회 API** 세 덩어리를 직접 만드는 것이다.

### 1.1 테마 문제는 두 갈래로 갈린다

| 영역 | 테마 처리 |
|---|---|
| **워크스페이스 셸** (WS10/WS20 툴바·트리·Attribute 패널 → HTML 컨버전 대상) | **새 CSS 변수 테마 시스템 필요** |
| **Preview iframe 내부** (`uiPreviewArea` 가 실제 UI5 bootstrap 로드, 사용자 앱 렌더) | **UI5 테마 그대로 유지** (진짜 UI5 가 돌아감) |

> 새 테마는 **셸 크롬용**만 만든다. 사용자가 디자인하는 앱 미리보기는 여전히 UI5 테마(`sap_horizon` 등)를 쓴다.

## 2. 기준 비주얼 = `sap_horizon`

테마의 **시각 언어는 UI5 `sap_horizon` 을 그대로 기준**으로 삼는다. 컨트롤의 모양·간격·모서리 반경·입체감(elevation)·폰트는 horizon 을 따른다.

> 이전에 검토한 `.themes/*.html` 커스텀 시안(Studio Light / Pastel Glass / Neo-Brutal / Windows 7·XP)은 **기준에서 폐기**한다. 그 시안들은 테마마다 모양 언어(radius·그림자·유리효과)가 제각각이라 구조가 발산했다. horizon 단일 기준으로 가면 **모든 테마가 동일한 모양을 공유하고 색만 달라지므로** 아키텍처가 단순해진다.

### 2.1 핵심 단순화 — "모양은 공통, 색만 테마별"

| 토큰 범주 | 테마 간 차이 |
|---|---|
| shape (radius / spacing / elevation / font) | **없음** — horizon 공통 고정값 |
| color (배경 / 텍스트 / 강조 / 경계 / 상태색) | **테마별로 다름** |

→ "**컴포넌트 CSS 1벌 + 테마별 색 토큰 세트**" 모델이 깔끔하게 성립한다. 모양이 전부 horizon 으로 같으니 컴포넌트 CSS(shell.css)는 한 번만 작성하고, 테마는 색 토큰만 갈아끼운다.

### 2.2 색값은 "추측하지 말고 horizon 에서 떠와라"

배경/텍스트/경계 등 중립색은 실제 horizon 파라미터를 덤프해 이식한다. 강조색만 테마별로 교체한다.

```js
// 현행 UI5 앱 콘솔에서 1회 — sap_horizon / sap_horizon_dark 파라미터 전체 덤프
["sap_horizon", "sap_horizon_dark"].forEach(t => {
  sap.ui.getCore().applyTheme(t);
  sap.ui.require(["sap/ui/core/theming/Parameters"], P =>
    console.log(t, JSON.stringify(P.get(), null, 2)));
});
```

→ `white` = `sap_horizon`(라이트) 이식, `dark` = `sap_horizon_dark` 이식. `purple/red/green` = `white` 의 중립색을 공유하고 **강조색 램프만** 교체.

## 3. 테마 카탈로그 (5종)

| 키 (`data-theme`) | 성격 | 배경 모드 | 강조색 | UI5 대응 |
|---|---|---|---|---|
| **`white`** (기본) | horizon 라이트 기본 | 라이트 | horizon 블루 | `sap_horizon` |
| **`dark`** | horizon 다크 | 다크 | 블루(다크 튜닝) | `sap_horizon_dark` |
| **`purple`** | horizon 라이트 + 보라 강조 | 라이트 | 퍼플 | `sap_horizon` recolor |
| **`red`** | horizon 라이트 + 레드 강조 | 라이트 | 레드 | `sap_horizon` recolor |
| **`green`** | horizon 라이트 + 그린 강조 | 라이트 | 그린 | `sap_horizon` recolor |

- **기본 테마 = `white`** (`:root` 기본값).
- `purple/red/green` 은 `white` 의 중립색(배경·텍스트·경계)을 공유하고 강조색 토큰만 다르다.
- 각 색 테마의 **다크 변형**(예: `purple-dark`)은 토큰 구조가 동일하므로 추후 배경 모드만 바꿔 손쉽게 확장 가능. 우선 범위는 위 5종.

## 4. 토큰 계약

### 4.1 3계층 구조

```css
/* Layer 1: Primitive — horizon 중립 팔레트 + 강조색 램프 */
:root{
  /* horizon neutrals (harvest 기준 초기값, 확정 전 제안) */
  --c-white:#ffffff; --c-bg-light:#f5f6f7; --c-text:#1d2d3e; --c-text-muted:#556b82; --c-line:#d9d9d9;
  --c-bg-dark:#12171c; --c-surface-dark:#1c2228; --c-text-dark:#eaecee; --c-line-dark:#2c3540;
  /* accent ramps */
  --c-blue:#0070f2;   --c-blue-strong:#0064d9;
  --c-purple:#7858ff; --c-purple-strong:#5d36ff;
  --c-red:#e8000d;    --c-red-strong:#c60017;
  --c-green:#188918;  --c-green-strong:#0e6b0e;
  /* horizon 상태색 (테마 공통) */
  --c-success:#107e3e; --c-warning:#e76500; --c-error:#e90b0b;
}

/* Layer 2: Semantic — 컴포넌트는 이것만 소비. shape 는 horizon 공통 고정 */
:root{
  /* color (테마별로 override) — 아래는 white 기본값 */
  --app-bg:var(--c-bg-light); --surface:var(--c-white); --surface-raised:var(--c-white);
  --text:var(--c-text); --text-muted:var(--c-text-muted); --line:var(--c-line);
  --accent:var(--c-blue); --accent-strong:var(--c-blue-strong);
  --success:var(--c-success); --warning:var(--c-warning); --error:var(--c-error);
  /* interaction states (체크리스트 B) */
  --hover-bg:rgba(85,107,130,.08); --active-bg:rgba(85,107,130,.16);
  --selected-bg:rgba(0,112,242,.10); --selected-text:var(--accent);
  --focus-ring:var(--accent); --disabled-bg:#f5f6f7; --disabled-text:#a9b4be; --disabled-opacity:.4;
  /* value states — UI5 ValueState (체크리스트 B/F) */
  --state-error:var(--c-error);   --state-error-bg:#ffeaea;
  --state-warning:var(--c-warning); --state-warning-bg:#fff8e5;
  --state-success:var(--c-success); --state-success-bg:#eafaf0;
  --state-info:var(--accent);     --state-info-bg:#e8f1ff;
  /* scrollbar (체크리스트 A) */
  --scrollbar-thumb:#c4ccd4; --scrollbar-thumb-hover:#a9b4be; --scrollbar-track:transparent; --scrollbar-size:0.75rem;
  /* overlay / popup (체크리스트 C) */
  --overlay-scrim:rgba(0,0,0,.45); --popover-bg:var(--c-white); --popover-shadow:0 0.5rem 2rem rgba(34,54,84,.20);
  /* form / text 표면 (체크리스트 A/F) */
  --placeholder:#8b97a3; --selection-bg:rgba(0,112,242,.20); --caret:var(--accent);
  /* link / icon (체크리스트 G) */
  --link:var(--accent); --link-hover:var(--accent-strong); --icon:currentColor; --icon-muted:var(--c-text-muted);
  /* shape — horizon 공통(테마 무관 고정) */
  --radius:0.5rem; --radius-sm:0.25rem;
  --elevation:0 0.125rem 0.5rem rgba(34,54,84,.10);
  --focus-ring-width:0.125rem; --focus-ring-offset:0.0625rem;
  --font:"72","72full",Arial,Helvetica,sans-serif;
  --motion:.15s; /* prefers-reduced-motion 시 0 (체크리스트 I) */
}

/* Layer 3: 테마 = 색 토큰만 override (data-theme 한 줄로 전환) */
[data-theme="dark"]{
  --app-bg:var(--c-bg-dark); --surface:var(--c-surface-dark); --surface-raised:#2a323a;
  --text:var(--c-text-dark); --text-muted:#a9b4be; --line:var(--c-line-dark);
  --accent:#1c93f2; --accent-strong:#5dc6ff;
  /* dark 는 색 표면도 함께 override (체크리스트 표면 누락 방지) */
  --hover-bg:rgba(255,255,255,.06); --active-bg:rgba(255,255,255,.12);
  --selected-bg:rgba(29,147,242,.20); --disabled-bg:#1c2228; --disabled-text:#6b7782;
  --state-error-bg:#3a1a1d; --state-warning-bg:#3a2e12; --state-success-bg:#10301d; --state-info-bg:#10283a;
  --scrollbar-thumb:#3a444e; --scrollbar-thumb-hover:#4d5a66;
  --overlay-scrim:rgba(0,0,0,.6); --popover-bg:#1c2228; --popover-shadow:0 0.5rem 2rem rgba(0,0,0,.5);
  --placeholder:#6b7782; --selection-bg:rgba(29,147,242,.30);
}
[data-theme="purple"]{ --accent:var(--c-purple); --accent-strong:var(--c-purple-strong); }
[data-theme="red"]   { --accent:var(--c-red);    --accent-strong:var(--c-red-strong); }
[data-theme="green"] { --accent:var(--c-green);  --accent-strong:var(--c-green-strong); }
/* white = :root 기본값 그대로 (override 불필요) */
/* purple/red/green 은 --accent 만 바꾸므로, --accent 를 참조하는 상태 토큰
   (--focus-ring/--selected-text/--link/--state-info/--caret)도 자동으로 따라감 */
```

> **규칙**: 컴포넌트 CSS(shell.css)는 색·radius·shadow 를 하드코딩하지 않고 `var(--*)` 만 소비한다. `purple/red/green` 은 강조색만, `dark` 는 배경·텍스트·경계까지 override 한다.

### 4.2 의미 토큰 계약 (확정 — 키 고정, 값은 harvest 이식)

| 토큰 키 | 범주 | 의미 | 테마별 override |
|---|---|---|---|
| `--app-bg` | color | 앱/페이지 배경 | dark |
| `--surface` | color | 바·패널·카드 표면 | dark |
| `--surface-raised` | color | 한 단계 위 표면 | dark |
| `--text` | color | 기본 텍스트 | dark |
| `--text-muted` | color | 흐린 텍스트 | dark |
| `--line` | color | 경계선 | dark |
| `--accent` | color | 강조/브랜드 | **white/dark/purple/red/green 전부** |
| `--accent-strong` | color | 주요 액션/hover | **전부** |
| `--success` / `--warning` / `--error` | color | 상태색 | horizon 공통(원칙적으로 불변) |
| `--hover-bg` / `--active-bg` | state | hover/pressed 배경 | dark |
| `--selected-bg` / `--selected-text` | state | 선택 항목 배경/텍스트 | dark / 강조색 따라감 |
| `--focus-ring` | state | 키보드 포커스 링 색 | 강조색 따라감 |
| `--disabled-bg` / `--disabled-text` / `--disabled-opacity` | state | 비활성 표면/텍스트 | dark |
| `--state-error` / `-warning` / `-success` / `-info` (+ `-bg`) | state | ValueState 4종 테두리/배경 | `-bg`는 dark |
| `--scrollbar-thumb` / `-thumb-hover` / `-track` / `-size` | surface | 스크롤바 | dark |
| `--overlay-scrim` | surface | 모달 backdrop | dark |
| `--popover-bg` / `--popover-shadow` | surface | 드롭다운·메뉴·툴팁 패널 | dark |
| `--placeholder` / `--selection-bg` / `--caret` | text | placeholder / `::selection` / 커서 | dark / 강조색 |
| `--link` / `--link-hover` | text | 링크 | 강조색 따라감 |
| `--icon` / `--icon-muted` | icon | 아이콘 색(`currentColor`) | — |
| `--radius` / `--radius-sm` | shape | 모서리 반경 | **불변(horizon 공통)** |
| `--elevation` | shape | 그림자/입체 | **불변** |
| `--focus-ring-width` / `--focus-ring-offset` | shape | 포커스 링 두께/오프셋 | **불변** |
| `--motion` | shape | 트랜지션 시간(reduced-motion 시 0) | **불변** |
| `--font` | type | 폰트 스택 | **불변** |
| `--bp-s` / `-m` / `-l` / `-xl` | layout | 브레이크포인트(UI5 Device 표준) | **불변(공통)** — 4.3 |
| `--space-*` / `--gutter` / `--pane-gap` | layout | 간격 스케일·유동 여백 | **불변(공통)** — 4.3 |
| `--tree-*` / `--attr-*` (min/basis/max) | layout | 3분할 패널 제약 | **불변(공통)** — 4.3 |
| `--hit-area` / `--row-h*` / `--control-h*` | layout | hit-area·밀도(터치 적응) | **불변(공통)** — 4.3 |

> 동일 의도의 색은 **단일 토큰**으로 통일한다(드리프트 금지). 강조색이 화면마다 미세하게 갈리지 않도록 `--accent` 하나로 모은다.  
> 상태색(`--focus-ring`/`--selected-text`/`--link`/`--state-info`/`--caret`)을 `var(--accent)` 로 참조해 두면 `purple/red/green` 에서 강조색만 바꿔도 **상태 표면이 자동으로 따라간다**. 이것이 **6.6 누락 방지 체크리스트**를 구조적으로 충족시키는 핵심이다.

### 4.3 반응형 토큰 (breakpoints · fluid scale)

> **반응형은 선택이 아니라 필수다.** UI5 `sap.m` 컨트롤(ResponsivePopover·OverflowToolbar·DynamicPage·`sap.ui.Device.media`)은 본래 반응형이었다. HTML5 로 교체하면서 이 반응형성을 **다시 직접 구현**해야 한다(7장). 토큰은 shape 와 마찬가지로 **테마 무관 공통값**으로 `:root` 에 1회 정의한다.

```css
:root{
  /* breakpoints — UI5 sap.ui.Device.media 표준(=horizon 기준). 값은 px 고정 */
  /* ⚠ CSS 변수는 @media 조건에 못 쓴다. @media 에는 아래 px 리터럴을 직접 쓰고,
     이 토큰은 JS(matchMedia)·container query·문서 단일 출처로만 사용한다. */
  --bp-s:600px;    /* < 600  : Phone(S)  — 패널 1개만, 나머지는 탭/오버레이로 접음 */
  --bp-m:1024px;   /* 600~1023: Tablet(M) — 2분할(트리+본문), 속성은 토글 패널 */
  --bp-l:1440px;   /* 1024~1439: Desktop(L) — 3분할 기본 */
  --bp-xl:1440px;  /* ≥ 1440 : LargeDesktop(XL) — 3분할 + 여유 간격 */

  /* fluid spacing scale — clamp() 으로 뷰포트에 따라 유동(테마 무관 고정) */
  --space-0:0; --space-1:0.25rem; --space-2:0.5rem; --space-3:0.75rem;
  --space-4:1rem; --space-6:1.5rem; --space-8:2rem;
  --gutter:clamp(0.5rem, 1.5vw, 1rem);          /* 컨테이너 좌우 여백 */
  --pane-gap:clamp(0.25rem, 0.6vw, 0.5rem);     /* 분할 패널 사이 간격 */

  /* 3분할 패널 제약 (스플리터 드래그 시 최소/기본/최대) */
  --tree-min:14rem;  --tree-basis:22%;  --tree-max:40%;
  --attr-min:16rem;  --attr-basis:26%;  --attr-max:44%;
  /* 중앙 미리보기 = 1fr (나머지 전부). 패널이 --*-min 아래로 줄면 접기 모드로 전환 */

  /* 터치/포인터 적응 — 마우스 vs 터치 hit-area (체크리스트 J) */
  --hit-area:1.75rem;            /* @media (pointer:coarse) 시 2.75rem 로 키움 */

  /* 밀도(density) — horizon Cozy 기본. Compact 모드는 행/패딩 축소 */
  --row-h:2.25rem; --row-h-compact:1.75rem;
  --control-h:2.25rem; --control-h-compact:1.75rem;
}

/* 포인터가 거친(터치) 환경: hit-area·행높이 확대 */
@media (pointer:coarse){
  :root{ --hit-area:2.75rem; --row-h:2.75rem; --control-h:2.75rem; }
}
```

> **규칙**:
> - `@media` 조건문에는 `--bp-*` 변수를 쓸 수 없으므로 **px 리터럴(600/1024/1440)을 직접** 쓰되, 그 값의 **단일 출처는 이 표**다(드리프트 금지). JS 분기는 `matchMedia(\`(min-width:1024px)\`)` 또는 `U4ATheme` 와 별개의 `U4ALayout` 헬퍼로 모은다.
> - 고정 px 보다 **유동 단위**(`%`/`fr`/`clamp()`/`min()`/`max()`)와 **container query**(패널 자체 너비 기준)를 우선한다. 데스크톱 창은 임의 크기로 리사이즈되고 스플리터로 패널 폭이 실시간 변하므로, 뷰포트 미디어쿼리만으로는 부족하다.
> - 간격·행높이·hit-area 도 색처럼 **토큰만 소비**한다. `shell.css` 에 `px` 매직넘버 간격을 하드코딩하지 않는다.

## 5. 전환 메커니즘 & API shim

### 5.1 `data-theme` 토글

```js
document.documentElement.dataset.theme = "purple"; // CSS 변수가 캐스케이드됨. 기본(미지정)은 white.
```

### 5.2 UI5 테마 API → 동일 시그니처 shim

호출부를 전부 고치지 말고, 같은 시그니처의 shim 으로 내부만 CSS 변수 기반으로 바꾼다(03 문서 `fnGetMsgClsText` 시그니처 유지 전략과 동일).

```js
const U4ATheme = {
  THEMES: ["white", "dark", "purple", "red", "green"],
  apply(name) {                                  // applyTheme() 대체
    const t = this.THEMES.includes(name) ? name : "white";
    document.documentElement.dataset.theme = t;
    window.dispatchEvent(new CustomEvent("u4a-theme-changed", { detail: { name: t } }));
  },
  param(varName) {                               // Parameters.get() 대체
    return getComputedStyle(document.documentElement).getPropertyValue(varName).trim();
  },
  onChange(cb) {                                 // attachThemeChanged 대체
    window.addEventListener("u4a-theme-changed", cb);
  }
};
```

> 기존 코드의 `u4a-ws-light-theme` 클래스 토글 자리에 `dataset.theme` 토글을 끼운다. 메타/레지스트리로 전달되던 UI5 테마명(`sap_horizon` 등)은 위 5종 키로 매핑한다(예: `sap_horizon→white`, `sap_horizon_dark→dark`).

## 6. AI 컨버전 실행 사양

> 이 절은 **설계 산출물**이다. 실제 구현은 추후 진행하며, 그 시점에 AI 는 이 절을 읽고 작업에 착수한다.  
> 토큰 **키(계약)** 는 확정 스펙으로 취급하고, 중립 색 **값** 은 `sap_horizon` 덤프(2.2)에서 이식한다. 강조색 램프만 테마별로 정의한다.

### 6.1 산출물 파일 트리 (target)

```
www/ws30/ws10_20/theme/
├─ tokens.css            ← Layer1 primitive + Layer2 semantic(:root=white 기본). shape=horizon 공통 (4.1)
├─ shell.css             ← 셸 공통 컴포넌트 CSS (var(--*)만 소비, 색·radius·shadow 하드코딩 0)
├─ theme-api.js          ← U4ATheme shim (apply / param / onChange) (5.2)
└─ themes/
   ├─ white.css          ← (기본은 :root, 필요 시 명시) sap_horizon 이식
   ├─ dark.css           ← [data-theme="dark"]  배경·텍스트·경계·강조 override (sap_horizon_dark 이식)
   ├─ purple.css         ← [data-theme="purple"] --accent 램프만 override
   ├─ red.css            ← [data-theme="red"]    --accent 램프만 override
   └─ green.css          ← [data-theme="green"]  --accent 램프만 override
```

### 6.2 토큰 계약 (4.2 표 = 확정. 키 고정, 값 이식)

모든 테마는 4.2 의 색 키를 정의하고, shape/type 키(`--radius*`, `--elevation`, `--font`)는 horizon 공통값으로 `:root` 에 1회 정의한다. `shell.css` 는 이 키만 소비한다.

### 6.3 컨버전 작업 순서 (각 단계 = 입력 → 산출 → 수용 기준)

1. **horizon 파라미터 harvest** — 입력: 현행 UI5 앱(2.2 덤프). 산출: `sap_horizon`/`sap_horizon_dark` 의 중립색·폰트·radius 값. 수용: white/dark 중립 토큰이 실제 horizon 값과 일치.
2. **토큰 작성** (`tokens.css`) — 입력: 4.1·4.2 + harvest 값. 산출: Layer1 primitive + Layer2 semantic(:root=white) + shape 공통. 수용: 4.2 모든 키 정의, shape 키는 테마 무관 1곳.
3. **컴포넌트 CSS 작성** (`shell.css`) — 입력: horizon 컨트롤 외형 + 셸 마크업. 산출: 색·radius·shadow 전부 `var(--*)`. 수용: `shell.css` 내 hex 색상/하드코딩 radius·shadow **0건**(grep 검증) **+ 6.6 누락 방지 체크리스트 표면 전수 토큰화**(스크롤바·드롭다운 펼침·포커스 링·상태·오버레이·placeholder 등).
4. **테마 색 세트 작성** (`themes/*.css`) — 입력: 3.테마 카탈로그. 산출: `dark`=배경·텍스트·경계·강조·**6.6 표면 색** override, `purple/red/green`=강조 램프만 override. 수용: `data-theme` 토글만으로 5종 전환, 모양은 전 테마 동일.
5. **테마 API shim** (`theme-api.js`) — 입력: 5.2 + 기존 `applyTheme/Parameters.get/attachThemeChanged` 호출부. 산출: 동일 시그니처 `U4ATheme.*`, UI5 테마명→5종 키 매핑. 수용: 기존 호출부 치환 후 동작 동일, 기본값 white.
6. **반응형 레이아웃 적용** (`shell.css` + 레이아웃 JS) — 입력: 4.3 토큰 + 7장 패턴. 산출: 3분할 유동 레이아웃·스플리터·OverflowToolbar·브레이크포인트(L→M→S) 전환·container query·팝업 재배치·터치 적응. 색·간격·폭은 전부 토큰(`--space-*`/`--tree-*`/`--attr-*`/`--hit-area`) 소비. 수용: 고정 px 폭 **0건**(grep), 창 리사이즈·스플리터 양끝 드래그·S/M/L/XL × 터치 모드에서 레이아웃 무손상.
7. **누락 표면 점검** — 입력: 6.6 체크리스트. 산출: 각 항목이 토큰을 소비하는지 5종 테마 × hover/focus/disabled/error/open 상태 **× S/M/L/XL 반응형 구간**으로 점검한 결과. 수용: 6.6 A~J 전 항목 통과(미적용 항목 0).

### 6.4 불변 제약 (반드시 보존)

- **기준 비주얼은 sap_horizon** — 모양(radius/elevation/spacing/font)은 horizon 공통 고정. 테마는 **색만** 바꾼다.
- **Preview iframe 내부는 UI5 테마 유지** — 셸 테마 토큰을 iframe 안으로 침투시키지 않는다(1.1).
- **컴포넌트 CSS 색 하드코딩 금지** — 반드시 토큰 소비(4.1).
- **`--accent` 등 동일 의도 색은 단일 토큰** — 드리프트 금지(4.2).
- **기본 테마 = white** — `data-theme` 미지정 시 white 로 동작.
- **텍스트는 토큰이 아닌 언어 함수로** — 화면 문구는 `oAPP.common.fnGetMsgClsText` / `parent.WSUTIL.getWsMsgClsTxt`(03 문서 7.17). 테마는 색/모양만 담당.
- **반응형은 필수** — 고정 px 폭/높이로 레이아웃을 못 박지 않는다. 유동 단위(`%`/`fr`/`clamp()`)·container query·4.3 브레이크포인트로 창 리사이즈·스플리터·터치에 대응한다(7장). UI5 가 제공하던 반응형(OverflowToolbar·ResponsivePopover·DynamicPage)을 직접 재현한다. **단, 미리보기 iframe 내부는 UI5 가 자체 반응형을 처리**하므로 침범하지 않는다.

### 6.6 테마 적용 누락 방지 체크리스트

> 디자인/테마 적용 시 가장 흔히 빠지는 표면들이다. 이유는 세 가지 — **① 정지 화면에 안 보임(상태/펼침) ② 브라우저·OS 렌더라 CSS 변수가 안 닿음 ③ '상태'라서 컴포넌트 매핑에서 누락**.  
> `shell.css` 와 각 테마는 아래 A~I 를 **전부** 토큰으로 커버해야 한다. (각 항목 옆은 필요한 CSS 문법 / 소비 토큰)

**A. 브라우저·OS 렌더 (별도 문법 필수, 변수 안 닿음)**
- [ ] 스크롤바 — `::-webkit-scrollbar*` + Firefox `scrollbar-color`/`scrollbar-width` (`--scrollbar-thumb/-track/-size`)
- [ ] placeholder — `::placeholder` (`--placeholder`)
- [ ] 텍스트 선택 — `::selection` (`--selection-bg`)
- [ ] 입력 커서 — `caret-color` (`--caret`)
- [ ] 자동완성 배경 — `input:-webkit-autofill`
- [ ] textarea 리사이저 / number 스피너 / search clear — `::-webkit-resizer` / `::-webkit-inner-spin-button` / `::-webkit-search-cancel-button`
- [ ] `<select>` 펼침 목록·checkbox·radio·range·date·file — 네이티브 한계 → `accent-color` 또는 **커스텀 컴포넌트로 교체**
- [ ] `<dialog>` backdrop — `::backdrop` (`--overlay-scrim`) / 리스트 마커 `::marker`

**B. 상태 (컴포넌트 아님 → 매핑에서 누락)**
- [ ] focus-visible 포커스 링 (`--focus-ring`, `--focus-ring-width/-offset`) — 접근성 필수
- [ ] hover / active(pressed) (`--hover-bg` / `--active-bg`)
- [ ] disabled / readonly (`--disabled-bg/-text/-opacity`)
- [ ] selected / checked / indeterminate (`--selected-bg/-text`)
- [ ] validation error/warning/success/information — UI5 `ValueState` (`--state-*`)
- [ ] loading/busy / expanded↔collapsed / dragging·drag-over(드롭 타깃)·drag ghost / empty / 링크 visited

**C. 오버레이·팝업 레이어 (정지 화면에 안 보임)**
- [ ] 드롭다운/콤보박스 **펼침 패널** + 항목 hover/selected/disabled (`--popover-bg/-shadow`)
- [ ] 컨텍스트 메뉴 / 서브메뉴 / 팝오버 / 툴팁(네이티브 `title`→커스텀)
- [ ] 모달 + backdrop(`--overlay-scrim`) / Toast·MessageBox·BusyDialog 오버레이
- [ ] Suggestion 드롭다운 / date picker 캘린더

**D. 컨테이너 크롬·부속**
- [ ] 구분선(`ToolbarSeparator`) / 오버플로 툴바 경계 / 오버플로(…) 버튼 + 팝업
- [ ] 스플리터·리사이즈 바 + hover (ServerList 좌우 분할)
- [ ] sticky 헤더 + 아래 그림자 / 스크롤 페이드 / 스켈레톤 / 스피너(`BusyIndicator`) / progress bar
- [ ] 빈 상태 일러스트(`IllustratedMessage`) / 탭(선택·indicator) / 아코디언 / 배지·태그

**E. 테이블·트리·리스트 (디자인 영역·서버리스트 핵심)**
- [ ] 컬럼 헤더 hover, 정렬 표시(asc/desc), sticky/frozen 컬럼
- [ ] row hover, 선택 row, zebra(교차색), 셀 포커스
- [ ] 트리 expand/collapse 셰브론, indent guide / 연결선
- [ ] 멀티선택 체크박스 컬럼, row 액션 hover, 그룹 헤더, no-data row, 컬럼 리사이즈 핸들

**F. 폼·입력 디테일**
- [ ] 라벨(필수 `*` 색), helper/description 텍스트
- [ ] 검증 메시지 텍스트 + 아이콘, 필드 value-state 테두리/배경 (`--state-*`)
- [ ] prefix/suffix(아이콘·단위), clear(x), 비밀번호 보기 토글, 글자수 카운터
- [ ] 스위치/토글 on/off + 핸들, 세그먼트 컨트롤, 스테퍼(+/−)

**G. 타이포·콘텐츠·아이콘**
- [ ] 링크 default/hover/visited (`--link/-hover`)
- [ ] 아이콘 색 `currentColor` 상속 — SVG fill 하드코딩 금지, 아이콘 버튼 hover 배경 (`--icon/-muted`)
- [ ] 상태 텍스트 색, disabled 텍스트, `<kbd>` 칩, `<mark>` 하이라이트

**H. Electron·데스크톱 특화 (이 앱)**
- [ ] 커스텀 타이틀바 버튼 hover (닫기=빨강) / 드래그 영역(`-webkit-app-region`)
- [ ] **비활성 창 디밍** — 포커스 잃은 창의 헤더/툴바 톤 다운

**I. 환경·모드 미디어쿼리 (거의 항상 누락)**
- [ ] `@media (forced-colors)` 고대비 모드 대응
- [ ] `prefers-reduced-motion` 시 트랜지션/애니메이션 끔 (`--motion:0`)
- [ ] 테마 전환 시 색 트랜지션(플래시 방지)

**J. 반응형·레이아웃 (필수 — 7장 상세)**
- [ ] **3분할 유동 레이아웃** — 트리·미리보기·속성을 `%`/`fr`/`clamp()` 로 구성, 창 리사이즈에 즉시 반영(고정 px 폭 금지). `--tree-*`/`--attr-*` min/basis/max 소비
- [ ] **스플리터 드래그** — 패널 폭 실시간 변경, `--*-min` 하한 도달 시 더 못 줄임 + 더블클릭 리셋. hover/active 토큰(`--hover-bg`/`--active-bg`)
- [ ] **브레이크포인트 전환** — L(≥1024) 3분할 → M(600~1023) 2분할(속성 토글 패널) → S(<600) 1패널 + 탭/오버레이. 4.3 기준값(600/1024/1440)
- [ ] **OverflowToolbar 재현** — 툴바 폭 부족 시 버튼이 오버플로(…) 메뉴로 접힘(`ResizeObserver` 또는 container query)
- [ ] **속성 패널 접기/펼치기** — 좁은 폭에서 오버레이/토글, sticky 헤더 유지
- [ ] **container query 우선** — 패널 내부 컴포넌트는 뷰포트가 아닌 **패널 자체 너비**(`@container`)에 반응(스플리터로 폭이 바뀌므로)
- [ ] **팝업/Popover 재배치** — 트리거 기준 위치 + 뷰포트 경계에서 flip/shift, 좁은 폭에서 전체폭 시트(sheet)로 전환
- [ ] **터치/포인터 적응** — `@media (pointer:coarse)` 시 hit-area·행높이 확대(`--hit-area`/`--row-h`), hover 의존 동작에 대체 경로
- [ ] **확대/축소·DPI** — 브라우저 zoom·OS 배율·고DPI 에서 깨지지 않게 `rem`/논리픽셀 기준, 미리보기 자체 zoom Slider 와 충돌 없음
- [ ] **오버플로·줄바꿈** — 긴 라벨/값 `text-overflow:ellipsis` + tooltip, 최소폭 이하에서 가로 스크롤 또는 wrap(잘림 금지)
- [ ] **콘텐츠 줌(reflow)** — `@media (min-resolution)` 아닌 `rem` 기반으로 400% 확대 시에도 가로 스크롤 없이 reflow(접근성 WCAG 1.4.10)

> **검증**: 정지 화면이 아니라 **창을 좁/넓게 리사이즈하고 스플리터를 양 끝까지 드래그**하면서 S/M/L/XL 각 구간 + 터치 모드에서 레이아웃이 무너지지 않는지 직접 확인한다. (정지 데스크톱 폭만 보고 끝내면 반응형 누락이 그대로 남는다.)

### 6.7 AI 실행 프롬프트

```text
너는 U4A Workspace 셸 크롬을 HTML5 로 컨버전하는 AI다. 테마 시스템을 구축한다.

참조 문서: .analy/12_테마_컨버전_전략.md (이 문서), .analy/03_메인프레임_WS10_WS20.md 7장·11장

원칙:
- 테마 = CSS Custom Properties. UI5 테마 컨셉을 CSS 변수로 대체한다.
- 기준 비주얼은 UI5 sap_horizon 이다. 모든 테마가 horizon 의 모양(radius/elevation/spacing/font)을 공유하고, 테마는 색만 다르다.
- 테마 카탈로그는 white(기본)/dark/purple/red/green 5종이다. white=sap_horizon, dark=sap_horizon_dark, purple/red/green=강조색 램프만 교체.
- Preview iframe 내부는 실제 UI5 가 돌아가므로 UI5 테마를 그대로 둔다. 셸 크롬만 컨버전 대상이다.
- 컴포넌트 CSS(shell.css)는 6.2 토큰 키만 소비하고 색/radius/shadow 를 하드코딩하지 않는다.
- 중립 색값은 추측하지 말고 sap_horizon/sap_horizon_dark 파라미터를 덤프해 이식한다. 같은 의도의 색은 단일 토큰(--accent)으로 통일한다.
- 테마 전환은 U4ATheme.apply(name) 로 하고, data-theme 미지정 시 white 로 동작한다.
- 흔히 빠지는 표면(스크롤바, 드롭다운 펼침 패널, focus-visible 링, hover/disabled/selected, ValueState, placeholder, ::selection, 모달 backdrop, 오버레이, 비활성 창 등)을 빠짐없이 토큰화한다. 6.6 체크리스트 A~J 를 전수 충족해야 한다. 정지 화면만 보고 끝내지 말고 hover/focus/disabled/error/open 상태를 직접 확인한다.
- 반드시 반응형으로 만든다. 고정 px 폭으로 레이아웃을 못 박지 말고, 유동 단위(%/fr/clamp())·container query·4.3 브레이크포인트(600/1024/1440)로 창 리사이즈·스플리터 드래그·터치(pointer:coarse)에 대응한다. UI5 의 OverflowToolbar/ResponsivePopover/DynamicPage 반응형을 직접 재현한다(7장). 미리보기 iframe 내부 반응형은 UI5 가 처리하므로 침범하지 않는다. 정지 데스크톱 폭만 보지 말고 창을 좁/넓게 리사이즈하고 스플리터를 양끝까지 드래그해 S/M/L/XL × 터치 모드를 직접 확인한다.

작업: 6.1 파일 트리를 6.3 순서대로(반응형 적용 포함) 산출하고, 각 단계의 수용 기준을 충족시킨다.
산출 후 6.4 불변 제약 위반이 없는지, 그리고 6.6 체크리스트 표면이 5종 테마 × 상태별 × S/M/L/XL 반응형 구간으로 전부 토큰을 소비하는지 self-check 한다.
```

## 7. 반응형(Responsive) 레이아웃 가이드

> **전제**: 이 앱은 **Electron 데스크톱 셸**이다. 따라서 반응형의 1차 대상은 "모바일 화면"이 아니라 **임의 크기로 리사이즈되는 창 + 스플리터로 실시간 변하는 패널 폭**이다. 그 위에 작은 창/태블릿/터치(pointer:coarse)/고배율·고DPI/콘텐츠 400% 확대까지 무너지지 않도록 한다. 모양 기준은 `sap_horizon`, 브레이크포인트는 UI5 `sap.ui.Device.media` 표준(4.3)을 따른다.

### 7.1 왜 다시 만들어야 하나 — UI5 반응형의 소실

| UI5 가 제공하던 반응형 | HTML5 에서 직접 재현 |
|---|---|
| `sap.m.OverflowToolbar` (폭 부족 시 버튼 자동 오버플로) | `ResizeObserver`/container query + 오버플로(…) 메뉴 |
| `sap.m.ResponsivePopover` (데스크톱=Popover, 폰=Dialog/Sheet) | 폭 분기로 Popover ↔ 전체폭 시트 전환 |
| `sap.f.DynamicPage` (스크롤 시 헤더 축소·sticky) | `position:sticky` + 스크롤 기반 헤더 축소 |
| `sap.ui.Device.media.attach` (S/M/L/XL 구간 이벤트) | `matchMedia` 구독 헬퍼(`U4ALayout`) |
| `sap.ui.layout.Splitter` (리사이즈 가능 분할) | flex/grid + 드래그 스플리터(7.3) |
| 컨트롤 내부 자동 wrap/truncation | `min-width:0` + `text-overflow` + tooltip(7.6) |

> UI5 컨트롤을 HTML 로 바꾸면 이 반응형 동작이 **함께 사라진다.** 컨버전은 외형뿐 아니라 이 동작들을 재현해야 "동등"이다(기능 인벤토리에 포함).

### 7.2 브레이크포인트별 셸 레이아웃 (3분할 → 적응)

| 구간 | 폭 | 디자인 영역(트리·미리보기·속성) 레이아웃 |
|---|---|---|
| **XL** | ≥1440 | 3분할. `--gutter`/`--pane-gap` 여유. 트리·속성 `*-basis`, 미리보기 `1fr` |
| **L** | 1024~1439 | 3분할 기본. 간격 축소 |
| **M** | 600~1023 | **2분할**: 트리+미리보기. 속성 패널은 우측 **토글 오버레이**(기본 접힘, 선택 시 슬라이드 인) |
| **S** | <600 | **1패널 + 탭/세그먼트**(트리 / 미리보기 / 속성 전환). 팝업은 전체폭 시트 |

```css
.design-shell{
  display:grid; gap:var(--pane-gap); padding:var(--gutter);
  grid-template-columns:
    minmax(var(--tree-min), var(--tree-basis))
    1fr
    minmax(var(--attr-min), var(--attr-basis));
  block-size:100%; min-block-size:0;  /* 자식 스크롤 위해 필수 */
}
.design-shell > *{ min-inline-size:0; }  /* 자식 truncation 허용(겹침·오버플로 방지) */

@media (max-width:1023px){            /* M: 속성 패널 접고 2분할 */
  .design-shell{ grid-template-columns: minmax(var(--tree-min),var(--tree-basis)) 1fr; }
  .design-shell .attr-pane{ position:fixed; inset-block:0; inset-inline-end:0;
    inline-size:min(var(--attr-max), 90vw); transform:translateX(100%);
    transition:transform var(--motion); box-shadow:var(--popover-shadow); }
  .design-shell.attr-open .attr-pane{ transform:none; }
}
@media (max-width:599px){             /* S: 1패널 + 탭 */
  .design-shell{ grid-template-columns:1fr; }
  .design-shell .tree-pane,.design-shell .preview-pane{ display:none; }
  .design-shell[data-active="tree"] .tree-pane,
  .design-shell[data-active="preview"] .preview-pane{ display:block; }
}
```

> 라우팅이 아니라 **CSS Grid 의 `grid-template-columns` 재정의**로 구간을 바꾸면 같은 DOM 을 유지한 채 레이아웃만 전환되어 상태(선택·스크롤)가 보존된다.

### 7.3 스플리터(리사이즈 핸들)

- L/XL 구간에서 트리·속성 패널 경계에 드래그 핸들을 둔다. 드래그로 `grid-template-columns` 또는 패널 `flex-basis` 를 갱신하되 **`--tree-min`/`--attr-min` 하한·`--tree-max`/`--attr-max` 상한**을 강제한다.
- 핸들 hover/active 는 `--hover-bg`/`--active-bg`, 폭은 `--hit-area`(터치 시 자동 확대). **더블클릭 → basis 리셋**.
- 드래그 중 `pointer` 이벤트 + `setPointerCapture`, 미리보기 iframe 위를 지날 때 이벤트 삼킴 방지 위해 드래그 동안 iframe 에 `pointer-events:none` 오버레이. (드래그 종료 시 해제 — 03 7.18 동기화와 무관, 호스트 레이아웃 전용)

### 7.4 OverflowToolbar 재현

```
[툴바 폭 측정(ResizeObserver/@container)]
  → 들어갈 버튼만 노출, 넘치는 버튼은 오버플로(…) 버튼 뒤 메뉴로 이동
  → 우선순위: Low(먼저 접힘) < High < Always(항상 노출=핵심 액션)
```
- container query 우선(`@container toolbar (max-width: …)`) — 툴바는 패널 자식이라 **뷰포트가 아니라 패널 폭**에 반응해야 한다.
- 오버플로 메뉴 패널은 `--popover-bg`/`--popover-shadow`, 항목 hover/selected/disabled 토큰 소비(6.6 C).

### 7.5 container query 우선 원칙

스플리터로 패널 폭이 뷰포트와 무관하게 변하므로, **패널 내부 컴포넌트는 뷰포트 미디어쿼리가 아니라 컨테이너 쿼리**로 반응한다.

```css
.attr-pane{ container-type:inline-size; container-name:attr; }
@container attr (max-width:22rem){
  .attr-row{ grid-template-columns:1fr; }   /* 좁으면 라벨·입력 세로 적층 */
}
@container attr (min-width:22rem){
  .attr-row{ grid-template-columns:9rem 1fr; }
}
```

### 7.6 오버플로·truncation·reflow

- 모든 flex/grid 자식에 **`min-width:0`** (없으면 긴 콘텐츠가 컨테이너를 밀어 깨짐 — 가장 흔한 버그).
- 긴 라벨/속성값: `overflow:hidden; text-overflow:ellipsis; white-space:nowrap` + 잘릴 때만 커스텀 tooltip(네이티브 `title` 대체, 6.6 C).
- 트리: 깊은 indent + 긴 노드명은 가로 스크롤 또는 ellipsis(노드 클릭 영역은 `--hit-area` 보장).
- **콘텐츠 400% 확대(WCAG 1.4.10 reflow)**: 모든 치수를 `rem`/논리단위로 → 확대 시 가로 스크롤 없이 reflow. 미리보기 자체 zoom(Slider)과는 별개 레이어.

### 7.7 터치·포인터·고DPI

- `@media (pointer:coarse)` 에서 `--hit-area`/`--row-h`/`--control-h` 확대(4.3). hover 로만 드러나던 행 액션·툴팁은 **탭/롱프레스 대체 경로** 제공.
- 고DPI/OS 배율: 1px 보더는 `1px` 유지(hairline) 하되 간격·폰트는 `rem`. 이미지·아이콘은 SVG 또는 `image-set()`.

### 7.8 레이아웃 전환 헬퍼 (`U4ALayout`)

`@media` 에 CSS 변수를 못 쓰므로(4.3), JS 분기는 단일 헬퍼로 모은다(테마의 `U4ATheme` 와 대칭).

```js
const U4ALayout = {
  Q: { s:"(max-width:599px)", m:"(min-width:600px) and (max-width:1023px)", l:"(min-width:1024px)" },
  current(){ return matchMedia(this.Q.l).matches ? "l" : matchMedia(this.Q.m).matches ? "m" : "s"; },
  onChange(cb){ Object.values(this.Q).forEach(q => matchMedia(q).addEventListener("change", () => cb(this.current()))); }
};
```

### 7.9 산출물 위치 & 검증

- 반응형 규칙은 `shell.css`(셸 공통) 또는 `design/css/common.css` 에 둔다. **별도 색·radius 토큰을 새로 만들지 않고** 4.3 레이아웃 토큰만 소비한다.
- 자가검증: `shell.css`/`common.css` 에 **고정 px 폭/높이 매직넘버 0건**(보더 hairline 제외) — grep. 그리고 창 리사이즈 + 스플리터 양끝 드래그 + S/M/L/XL + `pointer:coarse` 에뮬레이션 + 브라우저 400% 확대에서 **겹침·잘림·가로 스크롤 폭주가 없는지** 직접 확인.
