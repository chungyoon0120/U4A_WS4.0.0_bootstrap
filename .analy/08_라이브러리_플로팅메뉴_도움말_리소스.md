## 1. 개요

이 문서는 `www/lib/`, `www/floatingMenu/`, `www/help/`, `www/ext_api/`, `www/settings/`, `www/MSG/`, `www/ADMIN/`, `www/icons/`, `www/svg/`, `www/sound/` 영역을 분석한다.

해당 영역은 크게 세 가지 성격으로 나뉜다.

- **Electron/Node 전용 로직** (auto-update, PowerShell 실행, 워커, NSIS 훅 등): 앱이 Electron 데스크톱으로 유지되는 한 그대로 재사용
- **SAP UI5로 그려진 UI 레이어** (floatingMenu의 frame.js, ABAP.html 등): HTML5+바닐라 JS로 교체 대상
- **순수 리소스** (아이콘 GIF, SVG, WAV, MSG DB, JSON 등): UI5 여부와 무관하게 유지

```
├── SupportPackageChecker/
│   └── index.js                     ← SP(패치) 업데이트 오케스트레이터
```

### 2-2. electron-updater-sap.js — 메이저 버전 업데이트

**역할**: SAP 서버(`/zu4a_wbc/u4a_ipcmain/update_check`)에 현재 앱 버전을 POST하여, 서버 측 최신 버전이 더 높으면 업데이트 파일을 다운로드한다.

**핵심 동작 흐름**:

```
checkForUpdates(appVer, oServerInfo)
  └─ XHR POST → update_check (서버 버전 조회)
       ├─ RETCD=E → update-error-sap 이벤트 발행
       ├─ 서버 > 앱 버전 → update-available-sap + _getUpdateFileWorker(PARAM)
       │     └─ new Worker(majorUpdateWorker.js)
       │           └─ spawn(executor-node/index.exe, [--PRCCD=UPGRADE, ...])
       │                 ├─ stdout CHUNK_DOWN_OK → download-progress-sap
       │                 └─ close code=0 → update-downloaded-sap
       └─ 서버 ≤ 앱 버전 → update-not-available-sap

quitAndInstall()
  └─ REMOTE.app.relaunch({ execPath: __downPath })
  └─ REMOTE.app.exit(0)
```

**이벤트 목록** (document에 발행):

| 이벤트 이름 | 의미 |
|---|---|
| `checking-for-update-sap` | 버전 확인 시작 |
| `update-available-sap` | 업데이트 존재 |
| `update-not-available-sap` | 최신 버전 |
| `download-progress-sap` | 다운로드 진행 중 (TOTAL, COUNT) |
| `update-downloaded-sap` | 다운로드 완료 |
| `update-error-sap` | 오류 |

**v3.5.9 변경사항**: PowerShell 직접 호출 → `executor-node/index.exe` 방식으로 전환. PS 경로도 `[extraResource]/www/ext_api` → `[UserData]/ext_api`로 변경.

### 2-3. SupportPackageChecker/index.js — SP(패치) 업데이트

**역할**: 메이저 버전은 동일하지만 패치 번호(`SPLEV`)가 서버가 더 높을 경우, `app.zip`을 다운로드하고 `app.asar`를 교체한다.

**지원 방식**: SAP 서버 방식(`ISCDN=false`) / CDN(GitHub) 방식(`ISCDN=true`) 분기

```
checkForUpdates(remote, iscdn, versn, splev, oLoginInfo)
  ├─ iscdn=true  → gf_chkPatch_GIT() → Octokit GitHub API
  └─ iscdn=false → gf_chkPatch_SAP() → XHR POST /zu4a_wbc/u4a_ipcmain/WS_SUPPORT_PATCH (PRCCD=01)
       ├─ ISPATCH=false → update-not-available-SP
       └─ ISPATCH=true  → update-available-SP + _getPatchUpdateFileWorker(PARAM)
             └─ new Worker(supportPackageWorker.js)
```

**v3.6.3 추가**: `WSUTIL.closeAllMsgClsDb()` — 패치 시작 전 SQLite DB 연결 종료

### 2-4. majorUpdateWorker.js — 메이저 업데이트 워커

- Node `Worker` 컨텍스트에서 실행 (메인 스레드 블로킹 없음)
- `spawn(executor-node/index.exe, [--PRCCD=UPGRADE, --ACTCD=ws_update_file_get, ...])` 실행
- `stdout`에서 `CHUNK_DOWN_OK` 키워드 감지 시 카운터 증가 → 진행률 계산
- `close code !== 0` → `update-error-sap` 반환

### 2-5. supportPackageWorker.js — SP 업데이트 워커 (현행)

8단계 순서로 처리:

| 단계 | 내용 |
|---|---|
| Step 1 | executor-node.exe 로 `app.zip` 다운로드 (`--ACTCD=ws_support_patch_02`) |
| Step 2 | `ASAR.extractAll(app.asar → app/)` |
| Step 3 | `app.zip` → `app/www/` 에 압축 해제 |
| Step 4 | ExtraResources(`./www/MSG`) → resources/ 로 복사 |
| Step 5 | `app.zip` 삭제 |
| Step 6 | `node_modules.zip` 존재 시 압축 해제 |
| Step 7 | `ASAR.createPackage(app/ → app.asar)` |
| Step 8 | 임시 폴더 정리, `update-downloaded-SP` 발행 |

> `finally` 블록에서 임시 폴더·zip·`.wsx` 파일 일괄 정리.  
> v3.6.4: `_moveExtRes` — move→copy 방식 변경(원본 삭제 버그 수정).

### 2-6. _supportPackageWorker.js — SP 워커 구버전

`supportPackageWorker.js`의 이전 버전. `powershell.exe` 직접 호출 방식, `@electron/asar` 대신 `asar` 패키지 사용. 현재 활성 코드에서는 미사용이나 파일로 보존.

**유지 여부**: `lib/ws` 전체 — Electron 전용 로직이므로 **그대로 유지**.

## 3. lib/u4a — Iframe / PressTrigger / SessionWorker

### 3-1. 디렉터리 구조

```
www/lib/u4a/
├── ui/
│   └── Iframe.js              ← SAP UI5 커스텀 컨트롤
└── util/
    ├── PressTrigger-dbg.js    ← SAP UI5 커스텀 컨트롤 (디버그 소스)
    ├── SessionWorker.js       ← 순수 Web Worker (UI5 무관)
    └── SessionWorker-dbg.js   ← SessionWorker 디버그 소스 (내용 동일)
```

### 3-2. Iframe.js

- **정체**: `sap.ui.core.Control`을 확장한 UI5 커스텀 컨트롤 (`u4a.ui.Iframe`)
- **기능**: 브라우저 `<iframe>` 엘리먼트를 SAP UI5 렌더러를 통해 선언적으로 다룸 (src, width, height, border, align, frameborder 등 속성 제어)
- **컨버전 시**: UI5 컨트롤을 제거하고 순수 `<iframe>` 태그로 직접 교체 가능. 속성 제어 코드만 바닐라 JS로 재작성하면 됨.

### 3-3. PressTrigger-dbg.js

- **정체**: `sap.ui.core.Control`을 확장한 UI5 커스텀 컨트롤 (`u4a.util.PressTrigger`)
- **기능**: 지정된 시간(초) 동안 사용자 인터랙션(클릭/키업)이 없으면 `finished` 이벤트를 발행하는 타이머 컨트롤 (비표시 `<div>` 렌더링). 세션 타임아웃 감지 등에 활용.
- **컨버전 시**: `setInterval` + `addEventListener`로 동등 기능 구현 가능. UI5 의존도 없앨 수 있음.

### 3-4. SessionWorker.js

- **정체**: 순수 Web Worker (`self.onmessage`)
- **기능**: `keeptime`(분) 단위로 `setInterval`을 설정하고, 주기마다 `postMessage('X')`로 세션 유지 신호를 메인 스레드로 전송. `isActive=false`이면 인터벌 정지.
- **컨버전 시**: UI5 미사용이므로 **그대로 유지**.

## 5. 플로팅 메뉴 (floatingMenu)

### 5-1. 용도

SAP ABAP 에디터(SAPGUI)에 연동되는 독립 Electron `BrowserWindow`로, 오픈된 SAP 서버 탭 목록을 우측에 플로팅하여 표시. Source Patterns(USP), Code Templates(ABAP Editor), 개인화 탭 등을 포함.

### 5-2. 디렉터리 구조

```
www/floatingMenu/
├── index.html            ← 컨테이너(shell) — iframe 하나만 존재
├── frame.html            ← UI5 bootstrap 로드 후 frame.js 삽입
├── ABAP.html             ← ABAP Code Templates 탭 (UI5 bootstrap)
├── U4A.html              ← U4A 탭
├── USP.html              ← Source Patterns 탭
├── js/
│   ├── handle.js         ← BrowserWindow 생성/개폐/IPC 핸들러 (Electron API)
│   ├── fn_handler.js     ← IPC 이벤트별 처리 함수 (INIT/NEW_SERVER/CLOSE_SERVER/RECALL/ADDPESO/REMOVEPESO)
│   ├── frame.js          ← UI5로 전체 플로팅 메뉴 UI 구성
│   ├── code.js           ← 코드 관련 UI5 컨트롤
```

### 5-3. handle.js — BrowserWindow 생성

`exports.open(REMOTE, SCREEN, _DIRNAME, SSID)`:

- 이미 열려있으면 `webContents.send("IF-WS30-FLOARTMENU", {PRCCD:"NEW_SERVER"})` 후 포커스만
- 없으면 `transparent:true`, `frame:false`, 전체 화면 크기의 BrowserWindow 생성
- `floatingMenu/index.html` 로드 후 `{PRCCD:"INIT"}` 신호 전송
- `exports.close()`, `exports.send(chid, params)`, `exports.isOPEND()` 제공

### 6-2. u4a_helpdoc/main.js — 헬프 문서 다운로드·실행

**동작 흐름**:

```
exports.Excute(REMOTE, DOWN_ROOT_PATH, HELP_DOC_OPT)
  └─ gfn_getHeadData() → GET /zu4a_wbc/u4a_ipcmain/U4A_HELP_DOC_WS30?PRCCD=HEAD&LANGU_OUT=KO
       └─ _getHelpDocFileDown(PARAM)
             └─ new Worker(u4aHelpDownWorker.js)
                   ├─ OPEN_DOCU → _zipDocuOpen(options) / REMOTE.shell.openPath()
                   │     └─ openHelpDocViewer(options) → helpViewer.js (BrowserWindow)
                   ├─ SET_PROG_DATA → gfn_setProgressbar() (sap.ui.getCore().byId 사용!)
                   ├─ PROG_DIALOG_OPEN/CLOSE → oAPP.common.fnProgressDialogOpen/Close()
                   └─ ERROR → 오류 처리
```

**헬프 문서 포맷 변경 이력** (v3.6.0-3):
- 기존: `.chm` 파일 → `REMOTE.shell.openPath()` 실행
- 현행: `.zip` 파일 → 압축 해제 후 `helpViewer.js`(BrowserWindow)로 MkDocs HTML 렌더링

**진행바 처리에서 UI5 사용**:
```javascript
let oProgressbar = sap.ui.getCore().byId("u4aWsProg");
oProgressbar.setPercentValue(sPerValue);
```

### 6-3. helpViewer.js — 독립 뷰어 BrowserWindow

- `frame: false` (커스텀 타이틀바), `contextIsolation: false`, `nodeIntegration: true`

### 6-4. controllerClass · login · keyboard_shortcut

- **순수 HTML5+CSS** 작성 — UI5 미사용
- KO/EN 두 언어 각각 별도 폴더
- `keyboard_shortcut/img/`: WS10/WS20 워크스페이스별 단축키 안내 이미지 모음

### 9-1. 디렉터리 구조

```
www/MSG/
├── AJAX_REQ_MSG/
│   ├── KO/AJAX_REQ_MSG_001.json    ← AJAX 요청 중 표시 메시지 (한국어)
│   └── EN/AJAX_REQ_MSG_001.json    ← AJAX 요청 중 표시 메시지 (영어)
└── WS_COMMON/
    ├── KO/MESSAGE_CLASS.db         ← 공통 메시지 클래스 DB (SQLite, 한국어)
    └── EN/MESSAGE_CLASS.db         ← 공통 메시지 클래스 DB (SQLite, 영어)
```

### 9-2. 다국어 체계

**AJAX_REQ_MSG_001.json**: SAP 서버 AJAX 호출 중 로딩 인디케이터에 표시할 작업별 한/영 메시지.

```json

```

**MESSAGE_CLASS.db**: SQLite 포맷의 메시지 클래스 DB. `WSUTIL.getWsMsgClsTxt(언어, 메시지클래스, 번호)` API로 참조.  
- 메시지 클래스: `ZMSG_WS_COMMON_001` (WS 공통 메시지)
- 번호별 메시지: 업데이트 오류(290~310), 헬프 문서(367~372, 493~502) 등 500개 이상
- SP 패치(v3.6.3) 시 `WSUTIL.closeAllMsgClsDb()`로 DB 연결 종료 후 asar 교체

**참고**: `www/MSG/` 전체는 SP 패치 시 `extraResources`로 별도 관리되어 asar 외부에 배포된다.

## 10. ADMIN — 개발자도구 권한 제어

### 10-1. 구조

```
www/ADMIN/DevToolsPermission/
├── index.js          ← 권한 검증 및 DevTools 실행 로직
├── select.html       ← 파일 선택 UI (추정)
└── crypto-js.min.js  ← AES-128 복호화 라이브러리
```

### 10-2. 동작

배포 환경에서 개발자 도구(DevTools)를 열기 위해 암호화된 권한 파일이 필요한 보안 게이팅 모듈.

**두 가지 접근 방식**:

| 방식 | 함수 | 설명 |
|---|---|---|
| 파일 업로드 | `excute01(REMOTE)` | `.U4A_WS_DEVOPEN` 확장자 파일을 다이얼로그로 선택, AES-128 CBC 복호화 |
| 수기 입력 | `excute02(REMOTE, ENCRYPT)` | 암호화 문자열을 직접 전달하여 복호화 |

**검증 로직** (`fn_exCuteDevTools`):
1. 복호화된 JSON에서 `SYSID`, `EXPIRY_TIME` 추출
2. `EXPIRY_TIME < 현재년월일시분` → 만료 오류
3. `SYSID !== "*"` 이면 현재 접속 서버의 SYSID, 인스턴스 번호 일치 여부 확인
4. 통과 시 `REMOTE.getCurrentWindow().openDevTools()` 실행

암호화 키: `"U4A*RND*WS30"` (AES-128, CBC 모드)

| fn_handler.js IPC 처리 | `floatingMenu/js/fn_handler.js` | `getModel().oData`, `firePress()` 등 UI5 모델/이벤트 직접 조작 | DOM 기반 상태 관리로 전환 |
| 헬프 문서 진행바 | `help/u4a_helpdoc/main.js` (`gfn_setProgressbar`) | `sap.ui.getCore().byId("u4aWsProg")` | DOM `<progress>` 또는 CSS 진행바 |
| UI5 SDK 전체 | `node_modules/U4A/WS/lib/ui5/v11071/` | sap-ui-core.js 1.107.1 전체 | 교체 완료 후 제거 |

UI5 `sap.ui.getCore().applyTheme(...)` 호출은 `U4ATheme.apply(name)` shim(12번 5.2)으로 대체하고, 테마 전환은 `data-theme` 속성으로 캐스케이드한다(기준 `sap_horizon`, 테마 5종 white/dark/purple/red/green, 기본 white). 단, 플로팅 메뉴 ABAP 탭의 Monaco 에디터 테마(`lib/monaco/themes/*.json`)와 Preview 영역은 별도 체계로 그대로 둔다.

## 14. AI 컨버전 실행 사양 (플로팅 메뉴 / 도움말 뷰어 / lib UI 컨트롤)

> 이 절은 **설계 산출물**이며, 실제 구현은 추후 진행한다. 추후 "플로팅 메뉴/도움말까지 컨버전해" 요청 시, AI 는 이 절을 읽고 **기능 컨버전 + 테마 적용**까지 스스로 완결한다.  
> 기능 매핑은 5·6장과 13장(A 유지 / B 교체)을, 테마는 [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md) 를 따른다. 두 문서를 함께 적용해야 완성이다.  
> **적용 범위**: 이 문서가 다루는 영역 중 **UI(SAP UI5)를 가진 부분만** 컨버전 대상이다 — 플로팅 메뉴(`floatingMenu`), 도움말 진행바(`help/u4a_helpdoc/main.js`), `lib/u4a` UI5 커스텀 컨트롤. 그 외 리소스(아이콘/SVG/사운드/MSG/ext_api/settings JSON/ADMIN)와 Electron·Node 전용 로직(업데이트 워커, 헬프 뷰어 BrowserWindow, 헬프 다운로드 워커, SessionWorker, Monaco 번들)은 **순수 비UI**이므로 컨버전 대상이 아니며 그대로 유지한다(13장 (A)).

### 14.2.1 기능 동등성 — 기능 인벤토리 (전수 보존)

> **원칙**: 기존 기능·동작을 **하나도 빠짐없이 동등하게** 재현한다. 판단 기준은 외형이 아니라 **행동(behavior)**이다. UI5 컨트롤이 무엇이었는지가 아니라, 그 컨트롤이 **무슨 일을 했는지**를 보존한다.

컨버전 착수 전, 아래 카테고리별로 현재 화면의 모든 동작을 **전수 목록화**한다(누락 시 기능 손실로 간주).

| 카테고리 | 목록화 대상 |
|---|---|
| 이벤트 | click / dblclick / press / change / liveChange / submit 등 사용자·시스템 이벤트 전부 |
| IPC 처리 | `IF-WS30-FLOARTMENU` 채널의 `PRCCD` 분기(INIT/NEW_SERVER/CLOSE_SERVER/RECALL/ADDPESO/REMOVEPESO) 별 처리 결과 전부 |
| 메뉴·네비게이션 | 플로팅 메뉴 항목 동작, 탭 전환(IconTabBar→div 탭), NavContainer 페이지 이동, press 핸들러 |
| 상태 | Busy/Lock/disabled, 선택·토글 상태, 진행률(progress) 값, 개인화 목록 상태 |
| 입력·검증 | 개인화 항목 추가/삭제 입력값 검증, 빈 목록·중복 처리 |
| 엣지케이스·예외 | 다운로드 ERROR 분기, 빈 서버목록, 진행바 0/100 경계, 워커 메시지 누락 |
| 상호작용 | 자동완성/필터, 더블클릭, Iframe 속성 갱신, PressTrigger 타임아웃(finished) 발화 |

### 14.3 작업 순서 (각 단계 = 입력 → 산출 → 수용 기준)

1. **Electron/Node 자원 보존 확인** — 13장 (A) 의 자원(`handle.js` BrowserWindow/IPC, 헬프 뷰어 `helpViewer.js`, 다운로드 워커, SessionWorker, Monaco)은 **호출부를 그대로 유지**. UI 레이어만 교체. 수용: (A) 항목 코드 변경 0건, `IF-WS30-FLOARTMENU` IPC 채널명·`PRCCD` 분기 불변.
2. **UI5 → HTML 마크업 매핑 (플로팅 메뉴)** — 13장 (B) 대로 `sap.m.Page/Button/ResponsivePopover`·`IconTabBar`·`NavContainer` → HTML div/탭/리스트. 수용: `frame.html`/`ABAP.html` 에서 `sap-ui-bootstrap` 미삽입, 화면에 `sap.*` 컨트롤 미생성.
3. **UI5 → HTML 매핑 (도움말 뷰어 진행바)** — `gfn_setProgressbar` 의 `sap.ui.getCore().byId("u4aWsProg").setPercentValue(...)` → DOM `<progress>` 또는 CSS 진행바 갱신. 수용: 진행바가 UI5 참조 없이 동작, 다운로드 워커 메시지(`SET_PROG_DATA`) 흐름 불변.
4. **모델 → 상태/렌더 분리** — `fn_handler.js`·`frame.js`·`views/BASE/*` 의 `getModel().oData`·`firePress()` → JS 상태객체 + `render*()` 함수. 수용: 모델 변경 시 DOM 동기화 동작, IPC 이벤트(INIT/NEW_SERVER/CLOSE_SERVER/RECALL/ADDPESO/REMOVEPESO) 처리 결과 동일.
5. **테마 토큰 소비** — 모든 색·radius·shadow 를 `theme/tokens.css` 의 의미 토큰(12번 6.2)으로. 수용: `frame.css`/`codeTemp.css` 및 진행바 CSS 내 hex 색상·하드코딩 radius·shadow **0건**(grep 검증) **+ 12번 6.6 체크리스트 표면**(스크롤바·메뉴 드롭다운 펼침·focus 링·hover/disabled·placeholder·오버레이·progress bar 등) 전수 토큰화. Monaco 에디터 테마는 예외(별도 체계 유지).
6. **테마 전환 연동 + 텍스트 비하드코딩** — `applyTheme` 대신 `U4ATheme.apply(name)`(12번 5.2)로, 플로팅 메뉴 문서 루트에 `data-theme` 적용. 화면 문구는 메시지 키(`WSUTIL.getWsMsgClsTxt` / `MSG/` JSON, 9장)로 처리. 수용: 테마 전환 시 플로팅 메뉴·진행바 색 함께 변경, 하드코딩 문구 없음.

### 14.4 불변 제약 (반드시 보존)

- **기능 동등성** — 기존 기능·이벤트·IPC 처리·상태·엣지케이스를 빠짐없이 동등 재현(행동 기준). 임의 단순화·생략 금지.
