# 09. 환경 구축 런북 (AI 자동 실행용)

> **목적**: 이 문서만 보고 AI(또는 사람)가 **빈 폴더에서 Electron 설치 + npm install + 네이티브 모듈 빌드 + 실행**까지 스스로 수행할 수 있게 한다.
> 앱은 **Electron 데스크톱 앱으로 유지**되고, 화면(UI)만 SAP UI5 → HTML5로 교체된다. 따라서 **백엔드/Node 의존성은 원본과 동일하게 설치**한다.
> 대상 OS: **Windows x64**. 패키지 매니저: **npm** (lockfile 보존 시 `npm ci` 권장).

---

## 0. AI 실행 체크리스트 (이 순서대로)

```
[ ] 1. Node.js(빌드용) 설치 확인        → node -v / npm -v
[ ] 2. 작업 폴더 생성 후 이동
[ ] 3. package.json 작성 (아래 1절 전체를 그대로 기록)
[ ] 4. npm install                      → dependencies + devDependencies 설치(Electron 포함)
[ ] 5. npm run postinstall              → 네이티브 모듈(better-sqlite3) Electron ABI로 rebuild
[ ] 6. 설치 검증                        → 4절 명령
[ ] 7. (앱 소스 배치 후) npm start      → electron . 실행
```

## 2. 설치 명령 (AI가 셸에서 실행)

```bash
# (작업 폴더에서, 위 package.json 을 먼저 작성한 뒤)

# 1) 전체 의존성 설치 — Electron 14.2.9 + 백엔드 npm 패키지 전부 자동 설치
npm install

# 2) 네이티브 모듈(better-sqlite3)을 Electron 런타임 ABI로 재빌드
#    (package.json 의 postinstall 이 자동 실행되지만, 누락 시 수동 실행)
npm run postinstall
```

- `npm install` 한 줄로 **Electron 본체가 devDependencies 로 함께 설치**된다. Electron을 따로 글로벌 설치할 필요 없다.
- **lockfile이 있으면** 정확한 재현을 위해 `npm install` 대신 **`npm ci`** 를 사용한다(원본 `package-lock.json` 보존 권장, lockfileVersion 3 / 758 패키지).

### 네이티브 모듈 주의 (better-sqlite3)
- `better-sqlite3` 만 C++ 네이티브 모듈이다. **Electron이 내장한 Node ABI(Electron 14 → ABI 89)** 에 맞춰 컴파일돼야 앱에서 로드된다.
- `electron-builder install-app-deps`(= `npm run postinstall`)가 이를 자동 처리한다.
- 빌드 실패 시 Windows에 빌드 도구가 필요할 수 있다: Visual Studio Build Tools(C++ 워크로드) + Python. 그 경우:
  ```bash
  npx @electron/rebuild -f -w better-sqlite3
  ```

```bash
npm start          # = electron .  → electron/main.js → www/intro3.html 부팅
```

- 부팅 플로우 상세는 [01_부팅_아키텍처.md](01_부팅_아키텍처.md) 참조.
- 인스톨러(.exe) 빌드가 필요하면:
  ```bash
  npm run build    # postinstall(rebuild) + electron-builder --win --x64 → dist/
  ```

## 4. 설치 검증 (AI가 성공 여부 자가 확인)

```bash
# Electron 설치/버전 확인 (14.2.9 출력 기대)
npx electron --version

# 네이티브 모듈 바이너리 생성 확인 (.node 파일이 있으면 rebuild 정상)
#  ⚠️ `node -e "require('better-sqlite3')"` 로 검증하지 말 것!
#     이 모듈은 Electron ABI(89)로 빌드되므로 시스템 Node 에서 require 하면
#     NODE_MODULE_VERSION 불일치로 '실패'한다(설치 실패가 아님). 실제 로드는 앱(electron) 안에서만 정상.
ls node_modules/better-sqlite3/build/Release/better_sqlite3.node

# 핵심 의존성 존재 확인 (require.resolve 는 네이티브 로드를 하지 않으므로 안전)
node -e "['@electron/remote','electron-updater','regedit','ws','xml-js','better-sqlite3','electron'].forEach(m=>{try{require.resolve(m);console.log('OK',m)}catch(e){console.log('MISSING',m)}})"
```

- `npx electron --version` 이 `v14.2.9` 를 출력하면 Electron 설치 정상.
- `better_sqlite3.node` 파일이 존재하고 `electron-builder install-app-deps` 로그에 `finished moduleName=better-sqlite3` 가 찍혔으면 네이티브 rebuild 정상.
- better-sqlite3 의 **실제 런타임 로드 검증은 `npm start`(electron) 실행 시** 이뤄진다.

## 5. 런타임 버전 (참고)

| 항목 | 버전 | 메모 |
|------|------|------|
| Electron | **14.2.9** | devDependencies로 자동 설치 |
| └ 앱 내장 Node | 14.17.0 | Electron이 동봉. 앱 안의 Node API 기준 |
| └ Chromium | 93.0.4577.82 | |
| └ 네이티브 ABI | 89 | better-sqlite3 rebuild 타깃 |
| 빌드 머신 Node | 16+ 권장(현재 24.x 확인됨) | electron-builder 실행용. 앱 런타임과 별개 |
| electron-builder | ^26.0.12 | |

> ⚠️ 프런트엔드 번들 자산(Monaco/ACE/jQuery 등)은 **npm 설치 대상이 아니라** `www/` 폴더에 파일로 포함되는 것이라 환경 구축 단계와 무관하다. UI5는 컨버전으로 교체/제거 대상이므로 신규 환경에서 설치할 필요 없다.

`npm install` 로 들어오는 백엔드 패키지의 역할. **하나도 제거하지 않는다**(UI만 바꾸므로).

| `regedit` | Windows 레지스트리 읽기(SAP Logon landscape) |
| `xml-js` | `SAPUILandscape.xml` 파싱 |

| `electron-window-state` | 창 위치/크기 기억 |
| `electron-log` | 로깅 |
| `express` | 로컬 HTTP 서버 |
| `ping` / `systeminformation` / `node-screen` | 네트워크·시스템·화면 정보 |
| `puppeteer-core` | 개발용 헤드리스 브라우저 제어 |

| `dicer` / `arraybuffer-to-buffer` / `random-key` / `yamljs` / `http` / `net` | 보조 유틸 |

## 7. 컨버전 작업과의 관계

> 이 문서는 **환경 구축/실행 절차 런북**으로, UI5 → HTML5 컨버전 자체를 수행하는 문서가 아니다. 컨버전 작업의 **선행 환경 조건**(빈 폴더에서 Electron 설치 + npm install + 네이티브 모듈 빌드 + 실행)을 제공하는 메타 문서다.

- **실제 기능별 컨버전 실행 사양**은 각 기능 분석 문서(01~08)의 "AI 컨버전 실행 사양" 절을 따른다(예: [02_로그인_서버리스트.md](02_로그인_서버리스트.md) "## 9. AI 컨버전 실행 사양").
- **테마/색·모양 처리**는 단일 출처인 [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md)의 토큰 계약·전환 API·불변 제약을 따른다. 본 런북은 테마 산출물을 다루지 않는다.
- 따라서 이 문서에는 기능별 실행 사양 템플릿을 넣지 않는다. UI 교체는 위 기능 문서·테마 문서의 책임 범위다.

### 컨버전 이후 빌드/실행과의 연결점

- **백엔드/Node 의존성은 컨버전과 무관하게 그대로 유지**된다(6장). UI만 SAP UI5 → HTML5로 교체되므로, 컨버전 후에도 본 런북의 `npm install` → `postinstall`(better-sqlite3 rebuild) → `npm start` 절차는 변경 없이 그대로 적용된다.
- 컨버전으로 신설되는 프런트엔드 자산(테마 `tokens.css`/`shell.css`/`themes/*.css` 등)은 `www/` 폴더에 파일로 포함되므로 **npm 설치 대상이 아니며**(5장 주석 참조), 환경 구축 단계에 영향을 주지 않는다.
- 인스톨러(.exe) 빌드(`npm run build`)는 컨버전 후 교체된 `www/` 산출물을 그대로 패키징한다(1장 `build.files` 의 `www/**/*`). 컨버전 결과물이 이 경로 규칙을 따르는 한 빌드 파이프라인 수정은 불필요하다.
