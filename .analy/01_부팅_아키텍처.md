# 01. 부팅 & 전체 아키텍처 분석

> 작성 기준: 실제 소스 코드 직접 분석 (2026-06-09)  
> 대상 버전: U4A Workspace v3.6.3 (branch: analysis)  
> 컨버전 방향: UI 표현 레이어(SAP UI5)만 HTML5+CSS+바닐라 JS로 교체. Electron/Node 자원은 전면 유지.

---

## 1. 개요 (이 영역이 담당하는 책임)

이 영역은 앱이 실행되는 순간부터 사용자가 SAP 시스템에 로그인하여 메인 IDE 화면(WS10/WS20)이 완전히 초기화될 때까지의 **전체 부팅 파이프라인**을 담당한다.

구체적인 책임은 아래와 같다:

1. **Electron 네이티브 환경 초기화**: 싱글 인스턴스, 딥링크, 보안 설정, userData 경로 고정
2. **인트로(스플래시) 화면 표시**: 계절/월별 테마 기반 애니메이션 인트로
3. **초기 설치 작업**: userData 폴더 구조 생성, 레지스트리 키 초기화, VBS/PS 스크립트 복사
4. **설정 파일 병합 및 배포**: `ws_settings.json`을 런타임 환경 정보(경로, 버전, UI5 URL 등)와 합산하여 `conf/` 폴더에 배포
5. **서버 리스트 또는 로그인 화면 진입**: Trial 여부 및 글로벌 설정에 따라 분기
6. **메인 프레임(index.html) 로드 및 UI5 부트스트랩 동적 삽입**
7. **IPC 채널 등록 및 창 간 메타데이터 주입**
8. **SQLite 기반 메시지 클래스(WsMsgCls) 제공**

                  └─ 서버 선택 + 로그인 완료 시
                       └─ BrowserWindow(ws30/ws10_20/index.html) 열기
                            └─ webContents.send('if-meta-info', oMetadata)
                                 │
                                 ▼
[www/ws30/ws10_20/index.html]
  │  IPCRENDERER.on('if-meta-info') 수신
  │
  ├─ 메타데이터 전역 변수 등록 (setMetadata, setUserInfo, setSessionKey ...)
  ├─ window.require / module 임시 제거 (UI5 ↔ Node 충돌 방지)
  └─ UI5 bootstrap <script> 동적 삽입
       └─ data-sap-ui-oninit="_data_sap_ui_oninit"
            │
            ├─ _restoreNodeGlobals()     ← require / module 복원
            ├─ [로그인 없음] → _initRendering()
            │     └─ vw_main/view.js → vw_main/control.js
            │           └─ onInit() → _loadLoginPage()  ← Login/Login.html iframe 삽입

            └─ [로그인 있음] → library-preload.js 동적 삽입

### 2.2 단계별 조건 분기 정리

| 단계 | 조건 | 분기 목적지 |
|------|------|------------|
| intro.js 초기화 완료 후 | `oWsSettings.isTrial === true` | `fnTrialLogin()` → 직접 로그인 화면 오픈 |
| intro.js 초기화 완료 후 | `isTrial === false` (기본) | `fnOpenServerList()` → ServerList_v2 오픈 |
| ServerList에서 서버 선택 + 로그인 완료 시 | 항상 | `index.html` BrowserWindow 오픈, `if-meta-info` 전송 |
| `index.js` UI5 초기화 완료 (`_data_sap_ui_oninit`) | `getUserInfo()` 반환값 없음 | `vw_main/control.js` → Login iframe 삽입 |
| `index.js` UI5 초기화 완료 (`_data_sap_ui_oninit`) | `getUserInfo()` 반환값 있음 (새창) | `library-preload.js` 직접 주입 → WS 메인 화면 |

## 3. 파일별 역할

### 3.1 `electron/main.js`

**역할**: Electron 메인 프로세스 진입점. 앱 수명주기 전체를 관리.

| 함수 | 동작 |
|------|------|
| `app.setPath('userData', ...)` | `com.u4a_ws.app` 또는 `com.u4a_ws.app.dev` 경로로 userData 고정 |
| `new WsMsgClsService()` | `global.WsMsgCls` 싱글턴 생성; Renderer에서 `remote.getGlobal('WsMsgCls')`로 접근 |
| `remoteMain.initialize()` | `@electron/remote` 메인 초기화 (모든 창에서 remote 사용 가능) |
| `app.getAppPath` 재정의 | 경로 끝이 `www`가 아니면 `www`를 붙여 반환 |
| `configureRemoteDebuggingPort()` | 환경변수 `WS_REMOTE_DEBUG_HOST` 있을 때만 원격 디버깅 포트 설정 |
| `configureDeepLink()` | `u4a-ws` (빌드) / `u4a-ws-dev` (개발) 프로토콜 클라이언트 등록 |
| `configureSingleInstanceLock()` | 중복 실행 방지 (두 번째 실행 시 즉시 `app.quit()`) |
| `createWindow()` | `BrowserWindow` 생성 (frameless, 850x500, transparent, `intro3.html` 로드) |
| `configureSession()` | `SameSite=Strict/Lax` 쿠키를 `SameSite=None`으로 변환 (SAP 서버 호환) |
| `bindAppEvents()` | `whenReady`, `browser-window-created`, `web-contents-created` 이벤트 등록 |
| `attachBeforeInputEvent()` | Alt+F4 키 이벤트 가로채기 → `getOwnerBrowserWindow().close()` 호출 |

**핵심 WebPreferences 설정**:
```json
{
  "nodeIntegration": true,
  "contextIsolation": false,
  "webSecurity": false,
  "nodeIntegrationInWorker": true,
  "enableRemoteModule": true
}
```

**CommandLine 스위치 (주요)**:
- `disable-web-security` - CORS 비활성화
- `allow-file-access-from-files` - 로컬 파일 간 접근 허용
- `no-sandbox` - 샌드박스 비활성화
- `ignore-certificate-errors` - SSL 인증서 오류 무시

### 3.2 `electron/lib/msg/MessageDatabase.js`

**역할**: `better-sqlite3` 기반의 SQLite 파일 래퍼. 메시지 클래스 단건 동기 조회.

| 함수 | 동작 |
|------|------|
| `constructor(dbFilePath)` | 읽기 전용 모드로 SQLite 파일 오픈 + `MESSAGE_CLASS_TEXTS` 테이블용 prepared statement 초기화 |
| `getMessageClassRow(arbgb, msgnr)` | ARBGB(메시지 클래스 ID) + MSGNR(3자리 패딩) 으로 동기 조회. 반환: `{ ARBGB, MSGNR, TEXT, LTEXT }` |
| `close()` | DB 연결 명시적 종료 |

**DB 파일 경로**: `www/MSG/WS_COMMON/{LANGU}/MESSAGE_CLASS.db`

---

### 3.3 `electron/lib/msg/WsMsgClsService.js`

**역할**: `MessageDatabase` 인스턴스를 언어 코드별로 캐싱하는 서비스 클래스. Main 프로세스 전역 객체(`global.WsMsgCls`)로 등록되어 Renderer에서 `remote.getGlobal('WsMsgCls')`로 접근.

| 함수 | 동작 |
|------|------|
| `_resolveRootPath()` | 패키징 여부에 따라 `www` 루트 경로 반환 |
| `_getInstance(langu)` | 언어별 DB 인스턴스 캐시 조회/생성 |
| `getRow(langu, arbgb, msgnr)` | Renderer 호출 인터페이스 - 메시지 1건 동기 조회 |
| `close(langu?)` | 특정 언어 또는 전체 DB 연결 종료 |

### 3.4 `www/intro3.html`

**역할**: Electron 최초 로드 HTML. 스플래시 애니메이션(SVG 기반 U4A 로고 + 진행 상태 텍스트) 표시 후 `intro.js`를 로드.

### 3.11 `www/ws30/ws10_20/index.js`

**역할**: `if-meta-info` IPC 채널 수신 → 전역 변수 등록 → UI5 부트스트랩 동적 삽입.

| 함수 | 동작 |
|------|------|

## 5. IPC 통신 구조

### 5.1 IPC 채널 전체 목록

Renderer → Main → 전체 Renderer 브로드캐스트 방식으로 동작. `IPCMAIN`은 실제 `electron.ipcMain`을 `remote`로 참조하므로, Renderer 코드에서 `IPCMAIN.on()`을 직접 등록하는 특이한 구조를 사용한다.

| 채널 이름 | 방향 | 송신처 | 수신처 | 용도 |
|-----------|------|--------|--------|------|
| `if-meta-info` | Main → Renderer | `intro.js` / `ServerList.js` | `index.js` | 메인 프레임 창 초기화 메타 정보 전달 |
| `if-globalSetting-info` | Main → Renderer | `intro.js` | `ServerList.js` | 글로벌 설정(테마/언어 등) 전달 |
| `if-test-user` | Renderer → Main | `index.js (setUserInfo)` | Main (테스트용) | 유저 정보 저장 테스트 |
| `if-session-time` | Renderer → Main | `ws_fn_03.js` | `ws_fn_03.js` (IPCMAIN.on) | 세션 타임아웃 워커 시작 신호 |
| `if-browser-close` | Renderer → Main (broadcast) | `ws_events.js`, `ws_common.js` 등 | `ws_fn_ipc.js` | 브라우저 창 종료/세션 관리 |
| `if-exam-move` | Renderer → Main (broadcast) | `extopen.js` | `ws_fn_ipc.js` | 샘플 화면에서 WS 앱 이동 명령 |
| `if-dragEnd` | Renderer → Main (broadcast) | `ws_main.js` | `ws_fn_ipc.js` | Dialog 드래그 종료 이벤트 전파 |
| `if-send-action-{BROWSKEY}` | Renderer → Main → 특정 Renderer | `extopen.js`, 팝업들 | `ws_fn_ipc.js` | 브라우저별 액션 명령 (SETBUSYLOCK, BROAD_BUSY 등) |
| `if-browser-interconnection` | Renderer → Main (broadcast) | `ws_fn_04.js`, `ws_common.js` | `ws_fn_ipc.js` | 브라우저 간 데이터 공유/동기화 |
| `if-get-sys-info-{BROWSKEY}` | Main → 특정 Renderer | `ws_util.js` | `ws_fn_ipc.js` | 시스템 환경 정보 요청/응답 |

| `if-p13n-themeChange-{SYSID}` | Main → Renderer | (테마 변경 로직) | `ws_fn_ipc.js`, `Login.js` 등 | SYSID 기준 테마 변경 알림 |
| `if-browser-interconnection` | Renderer → Main (broadcast) | 여러 파일 | 모든 브라우저 | 창 간 데이터 메시지 브로드캐스트 |
| `{BROWSKEY}--find` | Main → 특정 Renderer | Find 팝업 | `fnFindPopupOpen.js` | 검색 팝업 결과 전달 |
| `{BROWSKEY}--errormsg--click` | Main → Renderer | 에러 팝업 | `ws_fn_ipc.js` | 에러 목록 행 클릭 이벤트 |

### 5.2 IPC 아키텍처 특성

- **`IPCMAIN`을 Renderer에서 직접 사용**: `@electron/remote`를 통해 `electron.ipcMain`을 Renderer가 직접 참조하여 `IPCMAIN.on()`으로 이벤트 구독. 이는 보안상 비권장이지만 이 앱의 핵심 통신 패턴.
- **브라우저 키(`BROWSKEY`) 기반 채널 네이밍**: 창별로 고유한 40자 랜덤 키를 붙여 특정 창 전용 채널 생성.
- **세션 키(`SESSKEY`) 기반 그룹화**: 같은 SAP 서버 세션을 공유하는 창들을 세션 키로 묶어 관리.
- **`BroadcastChannel` 보조 사용**: `extopen.js`에서 `BroadcastChannel("broadcast-to-child-window_${BROWSKEY}")`를 사용해 하위 팝업들에게 Busy 상태 동기화.

## 8. UI5 부트스트랩 메커니즘 상세

### 8.1 부트스트랩 삽입 흐름

UI5 부트스트랩은 `index.js`에서 `IPCRENDERER.on('if-meta-info')` 수신 시 동적으로 삽입된다.

```
IPCRENDERER.on('if-meta-info') 수신
    │
    ├─ 메타데이터 저장 (setMetadata, setUserInfo, setSessionKey, setThemeInfo ...)
    │
    └─ IIFE 실행:
        ① Node 환경 격리
           window.__node = { require, module, exports }
           window.require = window.module = window.exports = undefined
           (UI5 loader가 window.require를 덮어쓰는 것을 방지)
        
        ② ws_settings.json에서 UI5 설정 읽기
           let oSettings   = parent.WSUTIL.getWsSettingsInfo()
           let oBootStrap  = oSettings.UI5.bootstrap    // 기본 data-sap-ui-* 속성 맵
           let oThemeInfo  = parent.getThemeInfo()       // { THEME, BGCOL }
           let sLangu      = oSettings.globalLanguage    // 'KO' | 'EN' | ...
        
        ③ <script id="sap-ui-bootstrap"> 동적 생성
           - oBootStrap 객체의 모든 키 → setAttribute
           - data-sap-ui-theme      = 사용자 테마 (예: sap_horizon_dark)
           - data-sap-ui-language   = 글로벌 언어
           - data-sap-ui-libs       = sap.m, sap.tnt, sap.ui.table,
                                      sap.ui.layout, sap.f,
                                      sap.ui.codeeditor, sap.ui.unified
           - data-sap-ui-resourceroots = { "i18n_root": "./" }
           - data-sap-ui-oninit     = "_data_sap_ui_oninit"
           - src                    = oSetting_UI5.resourceUrl (로컬 UI5 번들)
        
        ④ document.head.appendChild(oScript)
           → UI5 로더가 비동기 로딩 시작
```

### 8.3 Node↔UI5 격리 이유 및 메커니즘

Electron Renderer 프로세스는 `nodeIntegration:true`이므로 `window.require`가 Node.js require를 가리킨다.  
SAP UI5 로더(`sap-ui-core.js`)도 모듈 시스템 탐지를 위해 `window.require`를 체크·오버라이드하는데,  
두 require가 충돌하면 Node 모듈 시스템이 파괴된다.

**해결책**: UI5 스크립트 삽입 직전에 Node globals를 `window.__node`에 백업하고 `undefined`로 만든 후,  
UI5 `attachInit` 완료 시점(`_data_sap_ui_oninit`)에 `_restoreNodeGlobals()`로 복구.

### 컨버전 진입 전략 (순서)

1. **`index.js` 수정**: `if-meta-info` 수신 후 UI5 Bootstrap IIFE 제거 → `_initRendering()` 직접 비동기 호출. Node↔UI5 격리 코드 제거.

3. **`vw_main/control.js` 교체**: UI5 라이브러리 로드 코드, EventBus, JSONModel → 바닐라 JS 객체·CustomEvent로 교체. `_loadLoginPage()`의 `sap.ui.core.HTML` → 직접 iframe 생성.
4. **`showMessage` 교체**: UI5 MessageToast/MessageBox → 자체 Toast/Dialog HTML 컴포넌트(재사용 가능한 util 함수).
5. **`setBusy` 교체**: UI5 `BusyDialog.open/close` → 이미 `#u4aWsBusyIndicator` DOM으로 전환된 `setBusy()`의 나머지 `oBusy.open/close` 코드 제거. `sap.ui.getCore().lock/unlock()` 제거.
6. **`library-preload.js` 수정**: `$.ajax({ async:false })` 기반 로드 → `<script>` 태그 순차 삽입 또는 `await import()` 체인으로 교체. jQuery 의존성 제거.
7. **CSS 테마 대체**: [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md) 의 토큰 계약(`theme/tokens.css`)을 소비하고, `ws_settings.json`의 `globalTheme` / 메타 `getThemeInfo().THEME` 값을 `U4ATheme.apply(name)`(12번 5.2) 으로 `data-theme` 에 매핑. UI5 테마 CSS 파일 제거. 색·radius·shadow 하드코딩 금지(토큰만 소비).

### 10.6 기능 동등성 — 기능 인벤토리 (전수 보존)

> **원칙**: 기존 기능·동작을 하나도 빠짐없이 동등하게 재현한다. 외형이 아니라 **행동**이 기준이다. UI5 컨트롤을 HTML 로 바꾸되, 그 컨트롤이 하던 **모든 동작**을 보존한다.

이 문서 본문(특히 2~8장)과 **실제 소스**(`index.js`, `vw_main/view.js`·`control.js`, `library-preload.js`, `ws_trycatch.js`, `intro.js`, `resources/index.js` 등)에서 아래 카테고리를 **전수 목록화**하라:

- **이벤트 핸들러** — 클릭/더블클릭/창 이벤트(`maximize`/`unmaximize`)/`online`·`offline`/`onAfterRendering` 콜백 등
- **키보드 단축키** — Alt+F4 가로채기, 전역 단축키(`globalShortcut`), `shortcut.js` 등록 키 등
- **입력·유효성 검증과 순서** — 필수값 체크 순서, ValueState 적용 순서, 분기 전 가드
- **화면 상태** — Busy(`#u4aWsBusyIndicator`/BusyDialog)·Lock(`lock/unlock`)·disabled(`setMainCloseBtnDisabled`)·valueState·네트워크 오류 레이어 표시/해제
- **서버 응답 분기** — IPC 수신 분기, 초기화 성공/실패 분기, 메타 유무 분기
- **엣지케이스·예외 처리** — 전역 에러 핸들러, 중복 호출 방지 flag, Trial 분기, 초기화 실패 시 errorpage
- **상호작용** — 컨텍스트 메뉴, 드래그(`-webkit-app-region: drag`·Dialog 드래그), 멀티선택, iframe 부모-자식 연동

**이 문서(부팅 아키텍처) 주제의 구체 예시**(문서/소스에서 실제 등장하는 동작):

1. **부트스트랩 로그인 전/후 분기** — `_data_sap_ui_oninit`(또는 그 대체 초기화)에서 `getUserInfo()` 유무로 Login iframe 삽입 vs `library-preload.js` 직접 주입(2.2 표, 8.2 체인)을 그대로 재현.
2. **`getUserInfo()` 분기** — `onInit()` 에서 `parent.CURRWIN.show()` 후 미로그인 → `_loadLoginPage()`, 로그인 → `loadWS30MainPage()` (3.13).
3. **전역 에러 핸들러 → 팝업 + 앱 종료** — `window.onerror`(`onError`) / `onunhandledrejection` → `DIALOG.showMessageBox` 후 `APP.exit()`, `showCriticalErrorDialog` 의 **중복 호출 방지 flag** 포함(3.17).
4. **BusyDialog / 전역 Busy 상태** — `setBusyDialog()`·`setBusy()` 와 `#u4aWsBusyIndicator` DOM 표시/해제, `sap.ui.getCore().lock/unlock()` 동등 동작(9장 B, 진입 전략 5).
5. **창 제어 / 최대화 토글** — 최소화·최대화·닫기 버튼 핸들러(`CURRWIN.minimize/maximize/close`)와 `maximize`/`unmaximize` 이벤트에 따른 최대화 버튼 아이콘 토글(`_attachCurrentWindowEvents`, 3.13).
6. **Alt+F4 가로채기** — `attachBeforeInputEvent()` 의 Alt+F4 → `getOwnerBrowserWindow().close()` 동작(3.1).
7. **IPC `if-meta-info` 수신 처리** — 메타데이터 전역 등록(`setMetadata`/`setUserInfo`/`setSessionKey`/`setThemeInfo` …) 후 셸 렌더 진입(2.1, 5.1, 8.1).
8. **isTrial 분기 / 인트로 설치 순서** — `setInitInstall` 콜백 후 `fnTrialLogin` vs `fnOpenServerList`, 인트로 설치 작업 시퀀스(2.2, 10.4 불변 제약).

**산출**: 아래 형식의 표로 보고한다.

| 기존 동작 | HTML5 구현 위치 | 검증 방법 |
|---|---|---|
| (예) `onunhandledrejection` → 팝업 + `APP.exit()` | (예) `index.js` 전역 핸들러 유지 | (예) 의도적 reject 발생 → 팝업·종료 확인 |

**수용 기준**: **인벤토리 항목 누락 0** — 위 카테고리의 단축키·이벤트·검증·분기·엣지케이스·상호작용을 전부 `기존 동작 → HTML5 구현 위치 → 검증 방법` 으로 매핑·검증한다. 매핑·검증되지 않은 항목이 하나라도 남으면 멈추고 보고한다.

### 10.3 작업 순서 (각 단계 = 입력 → 산출 → 수용 기준)

1. **Electron/Node 자원 보존 확인** — 9장 (A) 의 자원(`main.js`, `@electron/remote`, `IPCMAIN/IPCRENDERER`, `webContents.send`, `FS`/`PATH`/`SPAWN`, `regedit`, `better-sqlite3`(WsMsgCls), `pathInfo.js`, `ws_settings.json`, `userData` 구조, `CURRWIN.*`, `parent.*` 전역 체인 등)은 **호출부를 그대로 유지**. UI 레이어만 교체. 수용: (A) 항목 코드 변경 0건.
2. **UI5 부트스트랩 제거 + 마크업 교체** — `index.js`의 UI5 Bootstrap IIFE / Node↔UI5 격리(`window.__node`) 제거 → `_initRendering()` 직접 비동기 호출. `vw_main/view.js`의 `sap.m.App/Page/Bar/Image/Title/Button/VBox` → 9장 (B) 매핑대로 HTML DOM(`<div class="u4a-app">`, `<header>`, `<img>`, `<button onclick="parent.CURRWIN.*">` 등). `placeAt("content")` → `appendChild`. 수용: 화면에서 UI5 컨트롤(`sap.m.*`/`sap.ui.*`) 미생성, `if-meta-info` 수신 후 셸이 렌더됨.
3. **모델 → JS 상태객체** — `control.js`의 `sap.ui.model.json.JSONModel`/`getEventBus()`/`loadLibrary()` → 바닐라 JS 상태객체 + `CustomEvent`(또는 EventEmitter). `_loadLoginPage()`의 `sap.ui.core.HTML` 래핑 → `<iframe src="${sLoginPath}">` 직접 삽입. `showMessage`(MessageToast/Box)·`setBusy`(BusyDialog→`#u4aWsBusyIndicator`) 도 바닐라 컴포넌트로. 수용: 상태 변경 시 DOM 동기화, 로그인/WS30 분기(`getUserInfo()` 유무) 동작 유지.
4. **테마 토큰 소비** — 셸·Toast/Dialog·Busy 의 모든 색·radius·shadow 를 `theme/tokens.css` 의 의미 토큰(12번 6.2)으로. 수용: `main-shell.css` 내 hex 색상·하드코딩 radius·shadow **0건**(grep 검증) **+ 12번 6.6 체크리스트 표면**(스크롤바·focus 링·hover/disabled·ValueState·placeholder·오버레이·비활성 창 등) 전수 토큰화.
5. **테마 전환 연동** — `applyTheme`/`data-sap-ui-theme` 대신 `U4ATheme.apply(name)`(12번 5.2). `ws_settings.json`의 `globalTheme` 및 메타 `getThemeInfo().THEME`(8.1 부트스트랩 흐름) 을 `data-theme` 로 매핑. 수용: 테마 전환 시 메인 프레임 셸 색상 함께 변경, Preview/UI5 영역은 영향 없음.
6. **텍스트 비하드코딩** — 셸/메시지 문구는 기존 메시지 시스템(`WSUTIL.getWsMsgClsTxt`, `oAPP.msg.*`, 7장 메시지 클래스)을 그대로 사용. 수용: 화면 문구 하드코딩 없음(메시지 키 경유).

### 10.4 불변 제약 (반드시 보존)

- **Electron 부팅 시퀀스 전부 유지** — 9장 (A). 특히 `main.js`의 `createWindow`/`WebPreferences`/CommandLine 스위치, 인트로 설치 작업(`_saveWsSettingsInfo`·레지스트리·VBS/PS 압축 해제·패턴 JSON·로그 정리), `setInitInstall` 콜백 후의 `fnTrialLogin`/`fnOpenServerList` 분기.
- **IPC·메타 계약 유지** — `if-meta-info` / `if-globalSetting-info` 등 5장 채널명과 `oMetadata`(`SERVERINFO`/`THEMEINFO`/`EXEPAGE`/`SESSIONKEY`/`BROWSERKEY`) 구조를 바꾸지 않는다.
- **부팅 분기 흐름 불변** — `isTrial` 분기, `_data_sap_ui_oninit`(또는 그 대체 초기화 함수)에서의 `getUserInfo()` 유무 분기(Login iframe vs `library-preload.js`), `parent.*` 전역 공유 체인을 유지한다.
- **Node↔UI5 격리 코드는 UI5 제거와 동시에만 삭제** — UI5 부트스트랩이 남아 있는 동안에는 `window.__node` 백업/복구(8.3)를 임의로 제거하지 않는다(잔존 UI5 로더와 Node `require` 충돌 방지).
- **테마는 12번 문서 단일 출처** — 색/모양은 토큰만, 하드코딩·드리프트 금지. 메인 프레임 안에 로드되는 **Preview/UI5 영역(WS20 등)은 UI5 테마 유지**(셸 토큰 시스템을 iframe 안으로 침투시키지 않음, 12번 1.1). 인트로 스플래시의 계절 애니메이션 테마도 별개로 유지.
- **기능 동등성** — 기존 기능·이벤트·단축키·검증·상태·서버분기·엣지케이스를 하나도 빠짐없이 동등하게 재현한다(외형이 아니라 행동 기준). 임의 단순화·동작 생략 금지.
