## 1. 핵심 원칙 4가지

1. **한 번에 "작업 단위 하나"** — 각 기능 문서의 `## AI 컨버전 실행 사양` 절 하나가 완결 작업 단위다. "전부 컨버전해"가 아니라 "02 서버리스트 실행 사양대로 해"로 끊는다.
2. **항상 두 문서를 핀으로 고정** — *기능 문서 + [12_테마_컨버전_전략.md](12_테마_컨버전_전략.md)(테마)*. 디자인 영역이면 *[05](05_디자인영역.md) + [03](03_메인프레임_WS10_WS20.md) 7.17·7.18* 도. AI 에게 **"먼저 끝까지 읽고 시작하라"** 고 명시한다.
3. **임베디드 프롬프트를 베이스로** — 각 문서의 AI 실행 프롬프트 블록(예: 02 → 9.5, 12 → 6.7, 05 → 13.5)은 복붙용이다. "그 블록을 따르라"가 가장 짧고 정확하다.
4. **수용 기준으로 검수를 강제** — "작업 순서대로 하고, **각 단계 수용 기준에 대해 통과/실패를 표로 보고**하라"고 시키면 품질이 크게 오른다.

## 2. 작업 시작 전 "0단계" — 토큰 값 확보 (필수 선행)

12번 문서의 토큰 **값**은 아직 `harvest 기준 초기값(제안)` 이다. 그대로 두면 AI 가 색을 지어내 시안과 어긋난다. 그래서 **첫 작업은 항상**:

> "12번 2.2 의 덤프 스크립트로 `sap_horizon` / `sap_horizon_dark` 파라미터를 떠서 `tokens.css` 중립색을 확정하라."

— 이걸 먼저 시키거나, 사용자가 덤프 결과를 붙여준다. 이후 모든 화면이 같은 토큰을 공유하므로 흔들리지 않는다.

## 3. 재사용 지시문 템플릿

> 아래 `[목표]`의 문서/장 번호만 바꿔 그대로 사용한다.

```text

[먼저 읽기] 코드를 건드리기 전에 다음을 끝까지 읽어라:
- .analy/02_로그인_서버리스트.md (특히 8장 매핑 · 9장 실행 사양)
- .analy/12_테마_컨버전_전략.md (테마 토큰 · 6.6 누락 방지 체크리스트)

[진행 방식]
- 9.3 작업 순서를 그 순서대로 수행한다.
- 9.2 산출물 파일 트리에 명시된 파일만 만들거나 고친다.
- 모든 설계 결정은 문서의 절 번호로 근거를 대라. 문서에 없으면 추측하지 말고 나에게 물어라.

[가드레일 — 위반 금지]
- 기능 동등성: 기존 기능·이벤트·단축키·검증·상태·서버분기·엣지케이스를 하나도 빠짐없이 동등하게 재현한다(외형이 아니라 행동 기준). 먼저 기능 인벤토리(해당 문서의 "기능 동등성 — 기능 인벤토리" 절)를 만들고 각 항목을 구현·검증하라. 임의 단순화·동작 생략 금지.
- 9.4 불변 제약: Electron/Node 자원·인증/세션 흐름·oMetadata/IPC 계약을 그대로 유지. UI 레이어만 교체.
- 색은 theme/tokens.css 의 의미 토큰만 소비(하드코딩·드리프트 금지). 모양은 sap_horizon 공통.
- 반드시 반응형: 고정 px 폭 금지. 유동 단위(%/fr/clamp())·container query·12번 4.3 브레이크포인트(600/1024/1440)와 레이아웃 토큰으로 창 리사이즈·스플리터·터치에 대응(12번 7장). UI5 OverflowToolbar/ResponsivePopover/Splitter 반응형 직접 재현. 미리보기 iframe 내부는 침범 금지.
- 12번 6.6 체크리스트 A~J(스크롤바·드롭다운 펼침·focus 링·hover/disabled·ValueState·placeholder·오버레이·**반응형/레이아웃 J** 등)를 빠짐없이.
- 텍스트는 메시지 키(fnGetMsgClsText / WSUTIL.getWsMsgClsTxt)로, 하드코딩 금지.

[완료 보고]
- 기능 인벤토리 → `기존 동작 → 구현 위치 → 검증` 표 (누락 0)
- 9.3 각 단계 수용 기준 → 통과/실패 표

```

## 4. 반드시 짚어줄 가드레일 (안 넣으면 빠지는 것)

| 지시 문구 | 빠뜨리면 생기는 일 |
|---|---|
| **"기능을 하나도 빠짐없이 동등하게 — 기능 인벤토리 만들어 검증"** | 화면만 비슷하고 단축키·이벤트·검증·엣지케이스 동작이 절반 누락 |
| **"문서 절 번호로 근거를 대라"** | 문서 무시하고 자기 식대로 만듦 |
| **"색은 추측 말고 sap_horizon 덤프에서 이식"** | 색을 임의로 지어냄(시안과 어긋남) |
| **"12번 6.6 체크리스트 전수(A~J)"** | 스크롤바·드롭다운·포커스·상태 또 빠짐 |
| **"반드시 반응형 — 고정 px 폭 금지, 12번 7장대로"** | 정지 데스크톱 폭에서만 맞고 창 리사이즈·스플리터·좁은 폭·터치에서 레이아웃 붕괴 |
| **"불변 제약 위반 시 멈추고 보고"** | Electron/인증/세션 흐름을 무심코 뜯음 |
| **"문서에 없으면 물어라"** | 빈칸을 환각으로 채움 |
| **"산출물 트리에 있는 파일만"** | 범위 밖 파일을 건드려 회귀 유발 |

## 5. 흔한 실패와 방지

| 실패 패턴 | 방지 지시 |
|---|---|
| 외형만 맞추고 동작(단축키·검증·분기) 누락 | "**기능 인벤토리 전수 추출 → 매핑 → 검증**, 누락 0" 요구 |
| "horizon 스타일로 예쁘게" 같은 모호한 지시 | 항상 **특정 문서 + 실행 사양 절**을 지목 |
| 한 번에 여러 화면을 시킴 → 품질 저하 | **작업 단위 하나씩**, 끝나면 검수 후 다음 |
| 정지 화면만 보고 끝냄 | "hover/focus/disabled/error/open **상태를 직접 확인**" 명시 |
| 고정 px 폭으로 짜서 좁은 창·스플리터에서 붕괴 | "**반드시 반응형**(12번 7장)·고정 px 폭 금지, 창 리사이즈·스플리터 양끝·S/M/L/XL·400% 확대 직접 확인" 요구 |
| Preview iframe 까지 토큰 침투 | "**Preview iframe 내부는 UI5 테마 유지**" 못박기(05·03·12) |
| 텍스트 하드코딩 | "문구는 메시지 키로" 반복 명시 |
| 토큰 없이 색 하드코딩 | 완료 시 **`grep` 으로 hex 0건 자가검증** 요구 |

## 6. 권장 진행 순서 (11번 플레이북 8.4)

```
0. 토큰 값 확보 (sap_horizon harvest → tokens.css 확정)   [2절]
1. 테마 시스템 구축 (12번 6장: tokens → shell → themes/*.css → theme-api)
2. 02 로그인 / 서버 리스트
3. 01 · 03 셸(메인 프레임 + WS10/WS20 크롬)
4. 05 WS20 디자인 영역 (03 7.17·7.18 계약 준수)
5. 04 WS30/USP · 06 팝업 · 08 플로팅/도움말
```

각 단위를 끝낼 때마다 **기능 인벤토리(누락 0) + 수용 기준 + 12번 6.6 체크리스트**로 검수한 뒤 다음으로 넘어간다.

## 7. 한 줄 요약

> **"`.analy/<기능문서>` 의 'AI 컨버전 실행 사양' 절과 `.analy/12_테마_컨버전_전략.md` 를 먼저 끝까지 읽고, 작업 순서대로 수행하라. 먼저 기능 인벤토리를 만들어 기존 기능을 하나도 빠짐없이 동등하게 재현하고, 각 단계 수용 기준·기능 인벤토리(누락 0)·12번 6.6 체크리스트에 대해 통과/실패를 보고하라. 불변 제약 위반·문서에 없는 결정은 멈추고 물어라."**

이 한 문장 + 3절 템플릿이면 매번 짧게 지시할 수 있다.
